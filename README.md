# Devolution Idle Game

무서운 야생 몬스터를 공격해서 제압하는 대신, **순화**와 **교감**으로 팀에 합류시키는 턴제 수집 게임입니다.
몬스터는 전투와 성장을 거치며 점점 더 복잡해지는 것이 아니라, 오히려 **퇴화**를 통해 단순하고 귀여운 원형으로 돌아갑니다.

이 프로젝트의 핵심 목표는 아래 3가지를 동시에 만족시키는 것입니다.

* 공격 없는 턴제 전투가 실제로 재미있는가
* 무서운 몬스터가 귀여워지는 퇴화 보상이 강한가
* 팀 로테이션 기반 방치형 구조가 자연스럽게 성립하는가

---

## 1. Core Concept

기존 몬스터 게임의 기본 흐름:

* 싸운다
* 약하게 만든다
* 포획한다
* 진화시킨다

이 게임의 기본 흐름:

* 관찰한다
* 순화한다
* 인간이 직접 교감한다
* 팀에 합류시킨다
* 퇴화시킨다

즉 이 게임은 포켓몬처럼 **몬스터 대 몬스터 턴제 구조**를 유지하되, 공격 스킬 대신 **순화 / 안정화 / 교감 스킬셋**으로 싸우는 게임입니다.

핵심 차이는 최종 포획 단계가 아이템 투척이 아니라, **인간과 몬스터 사이의 교감 성립**이라는 점입니다.

---

## 2. High Concept Pitch

> "무서운 야생 몬스터를 아군 몬스터의 순화 행동으로 안정시키고, 인간이 직접 마지막 교감을 성립시켜 팀에 합류시키는 퇴화형 순환 방치 수집 로그라이트"

핵심 감정은 아래 3가지입니다.

1. 상대 반응을 읽는 재미
2. 교감 타이밍을 잡는 긴장감
3. 퇴화 후 귀여운 모습이 공개되는 기대감

---

## 3. Core Gameplay Loop

1. 야생 몬스터와 조우한다
2. 아군 몬스터의 순화 행동으로 상대를 안정시킨다
3. 상대의 반응을 읽으며 순화와 탈출을 관리한다
4. 적절한 순간 인간이 직접 교감 행동을 시도한다
5. 교감에 성공하면 몬스터가 팀 편성 가능 상태가 된다
6. 전투에 참여한 아군 몬스터는 경험치를 얻는다
7. 경험치가 찬 몬스터는 알 상태로 들어가 퇴화를 준비한다
8. 빈 자리를 새 몬스터로 메우며 팀을 순환 운영한다
9. 시간이 지나면 퇴화한 모습으로 복귀한다

---

## 4. Combat Overview

### 전투의 목표

이 게임의 전투 목표는 적 몬스터의 HP를 0으로 만드는 것이 아닙니다.

플레이어는 아래를 달성해야 합니다.

* 적 몬스터의 **순화 게이지**를 올린다 (UI에 게이지 바로 표시)
* 적 몬스터의 **도주 게이지**를 관리한다 (UI에 게이지 바로 표시)
* 적절한 타이밍에 **포획 계열 스킬**로 교감을 성공시킨다

### 3v1 전투 구조

한 턴에 **출전 중인 아군 3마리가 각각 1회 행동**하고, 적은 1회 행동합니다.

* 아군 3명이 동시에 출전하므로, 적의 일방적 공격 압박이 완화됨
* 각 몬스터가 역할을 분담하여 다양한 전략 조합 가능 (탱커/힐러/딜러)
* 포켓몬과 달리 교체 없이 3마리가 동시에 활동하는 구조

### 전투 결과

* **성공**: 적 몬스터와 교감 성립 → 팀 편성 가능 상태 획득
* **실패**: 적 몬스터 도주 (도주 게이지 초과)

### 전투 긴장감

적 몬스터는 순화만 받는 대상이 아니라, 전투 중 계속 아군 몬스터를 공격합니다.

* 적 공격 적중 시 아군 HP 감소
* HP 0이 되면 해당 몬스터는 **기절 상태**
* 기절 상태 몬스터는 일정 시간 또는 턴 동안 출전 불가
* 보유 몬스터 전부가 기절 상태가 되면 **게임 오버**

즉 이 게임은 평화로운 교감 퍼즐이면서도, 동시에 적의 공격을 버티며 타이밍을 잡아야 하는 **위험한 순화 전투**입니다.

## 5. Turn Structure

한 턴의 기본 구조는 아래와 같습니다.

1. **명령 페이즈**: 출전 중인 아군 몬스터 3마리가 **각각 1개 행동**을 선택한다
2. **속도 판정**: 민첩성 + 스킬 우선도에 따라 아군과 적의 행동 순서를 결정한다
3. **실행 페이즈**: 속도순으로 아군과 적이 **교차 실행**된다 (적이 중간에 끼어듦)
4. 각 행동에 대해 적 몬스터가 반응하고, 순화 / 도주 / HP / 감정 상태가 변한다
5. **턴 종료 페이즈**: 감정 상태 지속턴 감소, 어빌리티 효과 적용, 도주/순화 완료 체크
6. 다음 턴으로 넘어간다

아군과 적이 속도순으로 교차 행동하므로, **빠른 수비 캐릭이 먼저 방어 → 적 공격 → 느린 자극 캐릭 행동** 같은 전술적 흐름이 생기며, 민첩성 스탯이 전략적 의미를 가집니다.

## 6. Skill System — 3분류 체계

스킬은 3가지 계열로 분류되며, **가위바위보**처럼 상황에 맞는 선택이 핵심입니다.

### 자극 계열

적 몬스터의 **순화도를 상승**시키는 기본 행동입니다.

* 아군 몬스터의 다양한 감각 액션 (소리, 온도, 냄새, 행동)
* 적 몬스터의 **감각 타입 상성**에 따라 효과가 크게 달라짐
* 부가 효과로 적의 **감정 상태**(호기심, 매혹 등)를 유발할 수 있음
* 행동 우선도: 0 (중간)

### 포획 계열

적 몬스터와 **직접 교감**을 시도하는 행동입니다.

* 순화도가 높을수록 성공 확률 상승
* 실패 시 **도주 게이지 급상승** — 고위험 고보상
* 적절한 타이밍 판단이 핵심
* 행동 우선도: -1 (가장 느림)

### 수비 계열

적의 공격을 **방어하거나, 아군 체력을 회복하거나, 적 공격을 약화**시키는 행동입니다.

* 시간을 더 오래 벌어주는 역할
* 도주 게이지 감소 — 상황 안정화
* 부가 효과로 적에게 **진정** 상태를 유발할 수 있음
* 행동 우선도: +1 (가장 빠름)

### 스킬 자원 관리 — PP 시스템

각 스킬에는 **PP(사용 횟수)**가 있습니다.

* 강력한 스킬일수록 PP가 적음 (전격교감: PP 2, 빗소리: PP 8)
* 전투 내에서 PP를 소모하므로 스킬 선택에 긴장감이 생김
* PP는 전투 간 회복됨 (팀 화면에서)
* PP 부족 시 약한 기본 행동만 가능

### 스킬 풀 & 장착

* 각 아군 몬스터는 **8~12개의 보유 스킬 풀**을 가짐 (퇴화/성장으로 습득)
* 이 중 **3개를 장착**하여 전투에 사용
* 전투 전 팀 편성 화면에서 스킬 교체 가능
* 상대에 맞춰 스킬 조합을 바꾸는 것이 전략의 핵심

### 전략 구조

자극으로 순화도를 올리고 → 포획 타이밍을 노리되 → 도주 위험이 높으면 수비로 안정화.
적의 감정 상태와 감각 상성을 읽으며 3가지 계열을 적절히 조합하는 것이 전투의 핵심 재미입니다.

## 7. Stat System

### 6스탯 체계

포켓몬의 6스탯 구조를 순화 전투 패러다임에 맞게 재설계합니다.

| 스탯 | 역할 | 포켓몬 대응 |
|------|------|-------------|
| **친화력**(Affinity) | 자극 스킬 효율 (순화도 상승량) | 특공 |
| **공감력**(Empathy) | 포획 성공률 | 명중률 |
| **인내력**(Endurance) | HP, 받는 피해 감소, 수비 효율 | HP+방어 |
| **민첩성**(Agility) | 행동 순서, 회피 확률 | 스피드 |
| **유대력**(Bond) | 아군 시너지 보너스, 협동 효과 | 신규 |
| **직감**(Instinct) | 적 상태 읽기, 크리티컬 순화 확률 | 급소율 |

### 계산 공식

스킬 효과 = **스킬 위력 × (해당 스탯 / 기준값) × 상성 배율 × 감정 보정**

### 스탯 성장 — 행동 기반 노력치

* 전투 중 **사용한 행동 카테고리**에 따라 스탯 경험치가 축적됨
* 자극을 많이 쓰면 → 친화력 ↑, 수비를 많이 쓰면 → 인내력 ↑
* 포획을 많이 시도하면 → 공감력 ↑
* 플레이 스타일이 곧 몬스터의 성장 방향을 결정함

### 야생 상태 → 퇴화 상태

* **야생 (퇴화 전)**: 스탯이 분산됨 — 뭐든 조금씩 할 수 있지만 전문성 없음
* **퇴화 후**: 스탯이 1~2개에 집중됨 — 역할이 뚜렷해지고 전문성 상승
* 퇴화 시 **어빌리티(패시브 능력)**도 변화함

퇴화할수록 단순해지지만 **강해지는 구조**입니다.
이를 통해 팀 내 역할 분담(탱커/힐러/딜러)이 자연스럽게 형성됩니다.

## 8. Sensory Axes — 감각 상성 시스템

포켓몬의 타입 상성처럼, 감각 축 사이에 **순환 상성**이 존재합니다.

```
       소리(Sound)
      ↗          ↘
행동(Behavior)  온도(Temperature)
      ↖          ↙
       냄새(Smell)

상성: 소리 → 행동 → 냄새 → 온도 → 소리
유리: ×1.5 / 불리: ×0.5 / 보통: ×1.0
```

### 적 몬스터의 감각 타입

각 적 몬스터는 **감각 타입 1~2개**를 가집니다.

* 심연 늑대: 소리/행동 타입 → 냄새 자극에 약함, 온도 자극에 강함
* 유리 나방: 행동/소리 타입 → 온도 자극에 약함, 냄새 자극에 강함
* 바위 거북: 냄새/온도 타입 → 소리 자극에 약함, 행동 자극에 강함

### 상성의 의미

* **유리 상성**: 순화 효율 ×1.5, 도주 위험 감소, **호기심** 감정 유발 가능
* **불리 상성**: 순화 효율 ×0.5, 도주 위험 증가, **분노** 감정 유발 위험

플레이어는 적의 감각 타입을 보고 **어떤 아군/스킬을 쓸지 전략적으로 선택**합니다.
현재 MVP에서는 4축을 사용합니다.

* **청각**: 빗소리, 허밍, 정전기 소리
* **온도**: 따뜻함, 서늘함
* **후각**: 숲 냄새, 이끼 향
* **행동/거리**: 천천히 접근, 거리 유지, 숨기

## 8-1. Emotion System — 적 감정 상태

포켓몬의 상태이상(화상, 마비, 수면)을 **적 몬스터의 감정 변화**로 재해석합니다.

### 감정 상태 목록

| 감정 상태 | 효과 | 유발 조건 |
|-----------|------|-----------|
| **진정**(Calm) | 적 공격력 -50%, 도주 증가 멈춤, 2턴 지속 | 수비 스킬 부가 효과 |
| **호기심**(Curious) | 순화 효율 ×1.5, 2턴 지속 | 상성 유리 자극 히트 시 확률 발동 |
| **공포**(Fear) | 적이 매 턴 도주 +5, 공격력 +30% | 도주 게이지 80%+ 시 자동 전환 |
| **매혹**(Charmed) | 포획 성공률 +20%, 3턴 지속 | 특수 자극 스킬 부가 효과 |
| **분노**(Rage) | 적 공격력 ×2, 순화 효율 -50%, 2턴 지속 | 불리 상성 자극 연속 사용 |
| **신뢰**(Trust) | 도주 게이지 고정(더 안 오름), 순화 ×1.3 | 순화 70%+ 시 일정 확률 |

### 감정 상태의 전투 흐름 변화

감정 상태가 없을 때: 자극 자극 자극 → 포획 → 끝 (단조로움)

감정 상태가 있을 때: 자극(호기심 유발!) → 수비(진정 유지) → 자극(매혹!) → 포획 → 성공!

적이 **수동적 피격 대상**이 아니라, 감정이 변화하는 **살아있는 존재**가 됩니다.

## 8-2. Enemy AI — 성격 & 순화 단계별 행동

### 적 성격 시스템

각 적 몬스터에 **성격(personality)**을 부여하여 행동 패턴을 차별화합니다.

| 성격 | 특징 | 예시 몬스터 |
|------|------|-------------|
| **공격적**(Aggressive) | 매 턴 공격, 분노 유발 쉬움, 진정 저항 | 심연 늑대, 강철 멧돼지 |
| **겁많은**(Timid) | 도주 빠름, 대신 순화도 쉽게 오름 | 유리 나방, 메아리 박쥐 |
| **호기심**(Curious) | 가끔 공격 스킵(관찰), 상성 유리 시 큰 보너스 | 그림자 고양이, 수정 사슴 |
| **완고함**(Stubborn) | 순화 느림, 도주도 느림, 장기전 유도 | 바위 거북 |

### 순화 단계별 적 행동 변화

적은 순화도에 따라 행동이 달라집니다.

| 순화도 | 적 상태 | 행동 변화 |
|--------|---------|-----------|
| 0~30% | **경계** | 풀파워 공격, 높은 도주 증가 |
| 30~60% | **동요** | 공격력 감소, 가끔 행동 스킵 |
| 60~80% | **관심** | 공격 안 하는 턴 증가, 아군에 접근 |
| 80%+ | **교감 가능** | 거의 공격 안 함, 신뢰 상태 전환 가능 |
| 도주 80%+ | **공포** | 마지막 발악 강공 or 도주 시도 |

## 8-3. Ally Ability — 아군 패시브 능력

포켓몬의 특성처럼, 각 아군 몬스터에 **고유 패시브 능력** 1개를 부여합니다.

| 몬스터 | 어빌리티 | 효과 |
|--------|----------|------|
| 이슬요정 | **잔잔한 물결** | 매 턴 종료 시 도주 게이지 -2 (자동) |
| 숯뭉이 | **따뜻한 체온** | 인접 아군 HP +2 자동 회복 |
| 잎사귀요정 | **위장술** | 어그로 50% 감소, 기절 시 1회 HP 1로 버팀 |
| 수정벌레 | **수정 반사** | 방어 시 적에게 반사 데미지 2 |
| 이끼도롱 | **재생** | 매 턴 HP 10% 자동 회복 |
| 번개꼬리 | **축전** | 자극 성공 시 다음 자극 위력 +30% (스택 가능) |

퇴화 시 어빌리티도 변화합니다.
* 번개꼬리 → 전기콩: **축전** → **과부하** (자극 위력 +50%, 대신 본인 HP -3)
* 퇴화로 특성이 바뀌면서 전략이 변화하는 재미

---

## 9. Devolution System (기존 8번)

이 게임에서 성장은 더 복잡해지는 것이 아니라, 오히려 **퇴화**를 통해 단순하고 본질적인 형태로 돌아가는 것입니다.

### 퇴화 흐름

* 전투에서 행동한 아군 몬스터는 경험치를 얻는다
* 경험치가 임계치에 도달하면 **알 상태**에 들어간다
* 알 상태 몬스터는 전투에 출전할 수 없다
* 일정 시간이 지나면 퇴화가 완료되고 새로운 모습으로 복귀한다

### 퇴화의 의미

* 더 자주 사용한 몬스터일수록 먼저 빠진다
* 플레이어는 다른 몬스터로 팀 공백을 메워야 한다
* 기다림은 손해가 아니라, 퇴화 결과를 기대하는 시간이 된다

즉 방치형의 핵심은 플레이어를 막는 에너지 제한이 아니라, **몬스터가 퇴화를 위해 잠시 전장을 떠나는 시간 기반 순환 구조**입니다.

---

## 9. Idle / Rotation Structure

이 게임은 접속 중 플레이를 막지 않습니다.

* 플레이어는 원하면 계속 전투를 진행할 수 있다
* 별도의 행동력 제한은 없다
* 대신 핵심 전력 몬스터들이 알 상태로 빠지며 로테이션이 발생한다

### 결과적으로 생기는 전략

* 어떤 몬스터를 오래 쓸지
* 누가 먼저 알 상태로 들어갈지
* 새로 얻은 몬스터를 언제 팀에 넣을지
* 빈 자리를 누구로 메울지

즉 이 게임의 방치형 요소는 오프라인 골드 축적보다, **팀 전력 공백과 복귀를 관리하는 순환 구조**에 가깝습니다.

---

## 10. Team Structure

* 플레이어는 최대 **6마리**의 팀을 운영한다
* 이 중 **3마리가 출전**, 나머지 **3마리는 후보**로 대기한다
* 출전 3마리가 매 턴 동시에 행동하며, 역할을 분담한다
* 적 몬스터와 교감에 성공하면 팀 편성 가능 상태가 된다
* 빈 슬롯이 있으면 바로 편성 가능
* 빈 슬롯이 없으면 보유 상태로 유지하거나, 필요 없으면 자연으로 돌려보낼 수 있다

즉 수집은 무한 창고형보다, **순환형 로스터 관리**에 가깝습니다.

---

## 11. Run Structure & Meta Progression

이 게임은 한 번의 플레이를 하나의 **런(run)** 으로 볼 수 있습니다.

### 런 구조

* 초반에는 순화하기 쉬운 몬스터가 등장
* 점점 더 과도하게 진화된 몬스터가 등장
* 난이도가 높아질수록 순화 난이도와 공격 압박이 증가
* 결국 팀이 지치고, 훈미가 누적되며, 전멸하면 런 종료

### 게임 오버 이후

게임 오버가 완전한 상실은 아닙니다.

* 도감은 유지된다
* 해금한 몬스터 계열은 다음 시작 편성의 선택지가 된다
* 더 많은 몬스터를 수집할수록 시작 전략의 폭이 넓어진다

---

## 12. Start Team Rules

중요한 규칙은 다음과 같습니다.

* 야생 몬스터는 항상 **퇴화 전 상태**로 등장한다
* 따라서 스타트 팀도 기본적으로 **퇴화 전 야생 버전** 기준으로 선택한다
* 귀여운 퇴화형 자체를 스타트 유닛으로 고르는 것이 아니라, 그 계열의 야생 원본을 선택하는 구조다

### 메타 성장 방식

해당 몬스터의 퇴화 도감을 많이 해금할수록, 그 몬스터 계열의 **야생 버전 시작 스탯**이 강화됩니다.

예:

* 1단계 퇴화 해금 → HP +1
* 2단계 퇴화 해금 → 순화 스킬 위력 +1
* 3단계 퇴화 해금 → 기본 속도 +1
* 최종 퇴화 해금 → 추가 시작 특성 1개

즉 도감 완성은 단순한 수집 보상이 아니라, **다음 런의 시작 빌드를 강화하는 장기 성장**입니다.

---

## 13. Why This Can Be Interesting

이 구조가 매력적인 이유는 아래와 같습니다.

1. 공격 대신 돌봄이 핵심이라 차별화가 강함
2. 무서운 괴물을 이해하고 달래는 감정선이 강함
3. 퇴화 결과에 대한 시각적 기대감이 큼
4. 전투, 수집, 방치, 팀 운영이 하나의 순환으로 연결됨
5. 런 실패 후에도 도감과 시작 자유도가 남아 장기 동기가 유지됨

---

## 14. Critical Risks

냉정하게 보면, 이 게임은 독창적이지만 그대로 구현하면 **재미보다 복잡함이 먼저 올 위험**이 있습니다.

가장 큰 우려는 아래와 같습니다.

1. 전투에 관리해야 할 상태가 너무 많아질 수 있음
2. 순화 퍼즐이 고정 정답 암기 게임이 될 수 있음
3. 인간 교감이 마지막 확인 버튼처럼 느껴질 수 있음
4. 알 상태 로테이션이 전략보다 피로로 느껴질 수 있음
5. 힐링 컨셉과 로그라이트 실패 압박이 충돌할 수 있음

따라서 초기 개발 목표는 큰 시스템을 모두 구현하는 것이 아니라,
**공격 없는 순화 전투 1판 자체가 재미있는가**를 먼저 검증하는 것입니다.

---

## 15. MVP Goal

MVP의 목표는 전체 게임을 완성하는 것이 아닙니다.

아래 한 문장을 검증하는 것이 목표입니다.

> "적 몬스터의 반응을 읽고, 순화 행동을 고르고, 적절한 타이밍에 인간 교감을 넣는 1판 전투가 실제로 재미있는가?"

즉 MVP에서는 아래 3가지만 살면 됩니다.

1. 반응 읽기
2. 교감 타이밍 잡기
3. 퇴화 결과 기대하기

---

## 16. MVP Design Principles

1. 상태값을 줄인다
2. 감각 축을 줄인다
3. 인간 교감 행동 수를 줄인다
4. 분기와 예외 규칙을 줄인다
5. 한 판 전투를 3분 내외로 끝나게 한다
6. UI는 숫자보다 반응 로그를 중심으로 보여준다
7. 퇴화는 1회만 넣고, 감정적 보상만 검증한다

---

## 17. MVP Scope

### Include

* 팀 6마리 (출전 3 + 후보 3)
* 3v1 턴 구조 (속도순 교차 실행)
* 스킬 3분류 (자극/포획/수비) + PP 시스템
* 6스탯 체계 (친화력/공감력/인내력/민첩성/유대력/직감)
* 감각 4축 순환 상성 시스템
* 적 감정 상태 6종 (진정/호기심/공포/매혹/분노/신뢰)
* 적 성격 4종 + 순화 단계별 행동 분기
* 아군 어빌리티 (패시브 능력)
* 적 몬스터 8~10종
* 적 게이지 2개 (순화/도주, UI에 바로 표시)
* 아군 HP
* 전투 종료 후 경험치 획득 + 스탯별 노력치
* 경험치 임계치 도달 시 알 상태 진입
* 알 종료 후 퇴화 외형 + 어빌리티 변화 공개

### Exclude for Now

* 스킬 풀 8~12개 전체 (우선 몬스터당 4~5개로 시작)
* 순화 축 9종 전체
* 많은 퇴화 분기
* 세밀한 메타 보너스 종류
* 복잡한 장기 런 규칙

즉 MVP의 목표는 큰 게임을 만드는 것이 아니라,
**3v1 전투 1판 + 퇴화 1회 + 팀 교체 1회가 재미 루프로 성립하는지 확인하는 것**입니다.

---

## 18. MVP Combat Spec

### MVP 상태값

플레이어가 직접 보는 수치는 아래와 같습니다.

* **적 순화 게이지** — 적 정보 패널에 바로 표시
* **적 도주 게이지** — 적 정보 패널에 바로 표시
* **아군 HP** — 각 아군 정보 패널에 표시
* **무드 태그** — 순화/도주 비율에 따른 적 상태 키워드 (경계, 관심, 호감 등)

나머지 감정 상태는 내부 계산에 쓰더라도, 반응 로그(탄막)로 보조 표현합니다.

### MVP 스킬 구성 예시

각 몬스터는 3가지 스킬을 보유하며, 계열이 섞여 있습니다.

* 물 몬스터: `빗소리`(자극), `서늘한 안개`(수비), `물결 교감`(포획)
* 불 몬스터: `따뜻한 온기`(자극), `은은한 존재감`(수비), `불꽃 교감`(포획)
* 풀 몬스터: `숲 냄새`(자극), `숨기기`(수비), `부드러운 교감`(포획)

### MVP 적 예시

* 심연 늑대: 빗소리 선호, 따뜻함 선호, 강한 접근 비선호
* 유리 나방: 서늘함 선호, 허밍 선호, 손 내밀기 비선호

---

## 19. MVP Devolution / Idle Spec

* 행동한 몬스터는 턴 종료 시 경험치 +1
* 전투 종료 후 누적 경험치가 임계치 이상이면 알 상태 진입
* MVP에서는 몬스터당 1회만 퇴화한다
* 알 대기 시간은 짧게 설정한다
* 퇴화 후 외형 변화와 행동 1개만 바뀐다

핵심은 복잡한 성장 밸런스가 아니라,
**전투에 자주 쓴 몬스터가 빠지고, 다른 몬스터를 써야 하며, 기다리면 귀여운 모습으로 돌아온다**는 감정을 검증하는 것입니다.

---

## 20. MVP Validation Questions

MVP에서 반드시 확인해야 할 질문은 아래와 같습니다.

1. 3v1 전투에서 3마리 역할 분담이 전략적으로 느껴지는가?
2. 자극/포획/수비 3분류가 직관적으로 이해되는가?
3. 순화/도주 게이지 바가 전투 상황 파악에 도움이 되는가?
4. 포획 타이밍을 잡는 긴장감이 재미있는가?
5. 적 공격과 HP 압박이 수비 계열 사용 동기를 만드는가?
6. 알 상태 진입이 짜증보다 기대감으로 느껴지는가?
7. 퇴화 결과 공개가 다시 전투하고 싶게 만드는가?
8. 새 몬스터를 팀에 넣는 순간이 보상처럼 느껴지는가?

이 질문들에 긍정 답이 나오면, 그 다음에 로그라이트 구조, 메타 성장, 확장 감각 축을 붙여도 늦지 않습니다.

* 포획: 회수, 보호, 인도, 수용
* 진화: 퇴화, 복원, 원형 회귀, 정제
* 성장: 강화가 아니라 정리, 단순화, 순도 상승

## 22. Summary

이 게임은 몬스터의 순화 스킬로 야생 몬스터를 안정시키고 인간이 직접 교감을 성립시켜 팀 편성 가능 상태로 만든 뒤, 행동한 아군 몬스터들이 전투 결과와 무관하게 경험치를 쌓아 전투 종료 후 개별적으로 알 상태 퇴화를 판정받는 구조를 기반으로, 적의 공격과 훈미, 전멸, 런 리셋, 도감 기반 스타트 편성 확장을 함께 가진 **퇴화형 순환 방치 수집 로그라이트**입니다.

초기 개발에서는 이 모든 것을 한 번에 구현하기보다,
**공격 없는 순화 전투 1판이 실제로 재미있는지 검증하는 MVP**를 먼저 만드는 것이 가장 중요합니다.

## 23. Monster Generation Agent

몬스터 컨셉, 이미지, 게임 데이터를 자동 생성하는 멀티 에이전트 파이프라인입니다.

### 아키텍처

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ Concept Agent│────▶│ ComfyUI Agent│────▶│ Review Agent │────▶│  Candidate   │
│  (Qwen LLM)  │     │ (이미지 생성) │     │ (비전 심사)  │     │   Bundle     │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
   컨셉+스킬+            형태별 8장            베스트 선택          candidates/
   퇴화 트리 생성         이미지 생성           (단계별 기준)        폴더에 저장
```

### 에이전트별 역할

| 에이전트 | 모델 | 역할 |
|----------|------|------|
| **Concept Agent** | qwen3.5-35b-a3b (텍스트) | 몬스터 컨셉 생성: 이름, 설명, 감각 타입, 성격, 스탯, 스킬 3개×3 변종, 퇴화 트리, 적 반응 텍스트, 이미지 프롬프트 |
| **ComfyUI Agent** | z_image_turbo (이미지) | ComfyUI 워크플로우로 512×512 이미지 생성, 자동 배경 제거 포함 |
| **Review Agent** | qwen3.5-35b-a3b (비전) | 1:1 토너먼트 방식(8→4→2→1)으로 베스트 이미지 선택 |
| **Integration Tool** | qwen3.5-35b-a3b (텍스트) | 선택한 후보를 data.js 형식의 코드로 변환 + 에셋 복사 |

### 퇴화 트리 구조

```
기본형 (야생 적) ← 기괴하고 무서운 다크 판타지 스타일
  ├─ 퇴화1-A (아군) ← 강력하고 멋진 전투 크리처, 원본 흔적 보존
  │   ├─ 퇴화2-A1  ← 최대한 귀엽고 깜찍, 이전 단계 특징 유지
  │   └─ 퇴화2-A2
  ├─ 퇴화1-B (아군)
```
