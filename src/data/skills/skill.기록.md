# Skill Record

## 개요

- 위치: `src/data/skills/`
- 목적: 스킬 정의를 파일 단위로 분리하고, 몬스터 배분 전에 전체 스킬 목록을 표로 관리
- 현재 로딩 진입점: `src/data/skills/index.js`

## 파일 구조

| 파일 | 역할 |
|---|---|
| `core/schema.js` | 스킬 공통 스키마 정의, 기본값/복제 처리 |
| `core/registry.js` | 스킬 레지스트리 조립, 조회/복제 API 생성 |
| `catalogs/stimulate/*.js` | 자극 스킬 카탈로그 |
| `catalogs/capture/*.js` | 교감/포획 스킬 카탈로그 |
| `catalogs/defend/*.js` | 방어/회복 스킬 카탈로그 |
| `catalogs/index.js` | 카탈로그 묶음 정의 |
| `index.js` | 전체 스킬 레지스트리 병합 |

## 추가 규칙

| 항목 | 내용 |
|---|---|
| 기존 호환 | `makeActions(['sound-stimulate'])` 방식 그대로 유지 |
| 확장 방식 | 새 파일 추가 후 `index.js`에서 import + SOURCES 배열 등록 |
| 복제 처리 | 전투 중 PP 감소가 원본 데이터를 오염시키지 않도록 스킬 객체 복제 |
| 추천 다음 단계 | 몬스터별 `skillPool`, `equipped` 구조 추가 후 분배 시스템 연결 |

## 실제 배분 전에 필요한 최소 체크

| 체크 항목 | 이유 |
|---|---|
| 장착 스킬 3개가 `skillPool` 안에 있는가 | 데이터 오류 방지 |
| 마무리기 1개 이상 보유 여부 | 순화 게임의 승리 루프 보장 |
| 안정화기 또는 수비기 1개 이상 보유 여부 | 도주/광역 압박 대응 |
| 감정 유도 또는 감정 활용기 보유 여부 | 스킬 깊이 확보 |
| 퇴화 공백 시 대체 몬스터 존재 여부 | 장기 운영 가능성 확보 |

## 현재 설계 평가

| 항목 | 평가 |
|---|---|
| 기본 방향 | 적을 쓰러뜨리는 전투가 아니라 도주 전에 순화하는 구조와 잘 맞음 |
| 카테고리 구조 | `자극 / 교감 / 수비` 3분류는 명확하고 유지 가치가 높음 |
| 확장성 | 폴더 분리 + 레지스트리 구조로 확장성은 좋음 |
| 현재 한계 | 스킬 간 전술 차이가 아직 작고, 수치 차이 위주로 보이기 쉬움 |
| 부족한 요소 | 교체 연계, 감정 연계, 순화 단계별 마무리, 퇴화 로테이션 대응 |
| 종합 판단 | MVP 기본기 수준은 양호하지만, 로스터 운영형 메타를 만들기엔 추가 설계 필요 |

## 이 게임 기준 전투 구조

| 항목 | 현재/목표 해석 |
|---|---|
| 승리 조건 | 적 야생 몬스터가 도망가기 전에 순화 성공 |
| 전투 편성 | 주전 3마리 + 후보 3마리 |
| 교체 흐름 | 아군 HP 소진 시 후보가 전투 자리를 메움 |
| 포획 이후 | 팀 편성 투입은 별도 선택 사항 |
| 장기 운영 | 몬스터가 일정 레벨 도달 시 퇴화 진행 |
| 퇴화 페널티 | 일정 시간 사용 불가 |
| 방치형 연결 | 재접속 시 퇴화 완료 확인 가능 |

## 설계 개선 방향

| 개선 항목 | 이유 |
|---|---|
| 순화 가속기 강화 | 빠르게 순화 게이지를 올리는 역할이 명확해야 함 |
| 도주 억제기 분리 | 순화량은 낮아도 도주 게이지를 잡는 스킬 필요 |
| 감정 세팅기 추가 | `calm`, `curious`, `trust`, `charmed`를 전략적으로 만드는 축 필요 |
| 마무리 교감기 차별화 | 순화율이 높을수록 강해지는 피니시 스킬 설계 필요 |
| 교체 연계기 도입 | 후보 3마리 체계를 살리려면 교체 전후 가치가 생겨야 함 |
| 퇴화 대응 설계 | 특정 몬스터가 빠져도 다른 몬스터로 역할 대체가 가능해야 함 |

## 권장 스킬 역할군

| 역할군 | 설명 |
|---|---|
| 기본 자극기 | 안정적인 순화 게이지 상승, 기본 PP 효율 담당 |
| 고위험 자극기 | 순화량이 높지만 도주 위험도 큼 |
| 안정화기 | 도주 게이지 감소, `calm` 유도, 장기전 안정화 |
| 감정 유도기 | `curious`, `charmed`, `trust` 등 상태를 세팅 |
| 교감 마무리기 | 순화율이 높을수록 가치가 커지는 피니시 기술 |
| 수비/버티기 | 아군 생존, 다음 턴 확보, 후보 진입 전 시간 벌기 |
| 교체 연계기 | 퇴장 직전, 교체 직후, 혹은 다음 아군을 위한 판 정리용 |

## 포켓몬식 설계에서 가져올 것과 버릴 것

| 구분 | 적용 방향 |
|---|---|
| 가져올 것 | PP, 역할 분화, 상태이상 연계, 조건부 고성능 기술 |
| 부분 적용 | 위력 차등, 명중 대신 도주 위험/조건부 실패로 치환 |
| 버릴 것 | HP 0 중심 화력 메타, 타입 상성 중심 단순 딜 경쟁 |
| 우리식 변환 | `타입 메타` 대신 `감각 축 + 감정 상태 + 도주 관리` 메타로 전환 |

## 전투 상태별 우선 행동 규칙

| 전투 상태 | 우선 판단 | 권장 스킬군 | 기획 의도 |
|---|---|---|---|
| 전투 초반, 순화 낮음, 도주 낮음 | 적 감각축 확인 후 안정적으로 순화 시작 | 기본 자극기, 감정 유도기 | 초반엔 무리한 마무리보다 판 세팅이 중요 |
| 전투 초반, 도주가 예상보다 빠름 | 순화보다 먼저 도주 억제 | 안정화기, 수비/버티기 | 패배 조건이 도주이므로 위험 관리 우선 |
| 순화 30~60% | 중반 운영 구간, 감정 상태 세팅 시작 | 감정 유도기, 고위험 자극기 | 이 구간에서 콤보 기반 격차가 생김 |
| 순화 60~80% | 마무리 준비, 실패 리스크 계산 | 안정화기, 교감 마무리기 | 교감기 남발보다 성공 기대값 관리가 중요 |
| 순화 높음, 도주 낮음 | 순화 완료 시도 | 교감 마무리기 | 승리 턴을 당기기 위한 피니시 구간 |
| 순화 높음, 도주 높음 | 실패하면 패배 가능성이 큼 | 안정화기 후 교감, 혹은 고확률 마무리기 | 욕심내다 놓치지 않게 안전선 확보 |
| 아군 1마리 위험 상태 | 즉시 생존 확보 | 수비/버티기, 교체 연계기 | 다음 행동권과 후보 진입 가치를 보장 |
| 후보 투입 직후 | 새 아군이 판을 이어받게 해야 함 | 기본 자극기, 감정 연계기, 교체 연계기 | 교체가 손실이 아니라 전술 전환이 되게 설계 |

## 감정 메타 정의

| 감정 | 전투적 의미 | 권장 후속 행동 | 금지/주의 행동 |
|---|---|---|---|
| `calm` | 공격성과 도주 압박을 낮추는 안정 상태 | 교감 마무리기, 중위험 자극기 | 굳이 도주를 더 자극하는 고위험 연타는 비효율 |
| `curious` | 순화 효율을 높이는 탐색 상태 | 기본 자극기, 고효율 자극 콤보 | 바로 수비만 반복하면 기대값 손실 |
| `charmed` | 교감 성공률과 마무리 가치를 올리는 상태 | 교감 마무리기, 조건부 피니시 | 낮은 가치 기본기 반복 |
| `trust` | 장기적으로 가장 이상적인 순화 우세 상태 | 안정 교감, 저위험 마무리 | 불필요한 고위험 기술 사용 |
| `rage` | 적이 거칠어지고 순화 효율이 나빠진 위기 상태 | 안정화기, 수비/버티기, 감정 해제기 | 고위험 자극기, 무리한 교감 시도 |
| `fear` | 도주 압박이 매우 큰 붕괴 직전 상태 | 도주 억제기, 긴급 안정화기 | 느린 세팅기, 가치 낮은 회복 반복 |

## 감정 상태 시스템 원칙

| 항목 | 기준 |
|---|---|
| 감정의 위상 | 감정은 부가효과가 아니라, 순화 속도와 도주 제어를 바꾸는 핵심 전투 엔진으로 취급 |
| 좋은 설계 방향 | “상태를 건다”보다 “상태를 활용한다” 쪽의 스킬 비중을 늘린다 |
| 밸런스 원칙 | 강한 감정 부여 스킬은 PP, 조건, 도주 위험 중 하나 이상으로 제약 |
| 몬스터 차별화 | 어떤 몬스터는 `calm` 전문가, 어떤 몬스터는 `curious -> charm` 연계 전문가가 되게 설계 |
| 실패 구간 보완 | `rage`, `fear` 복구 전용 기술이 있어야 전투가 단순 리셋이 되지 않음 |

## 교체 연계 설계 기준

| 상황 | 필요 설계 | 이유 |
|---|---|---|
| 퇴장 직전 | 상태를 남기거나 도주를 낮추는 기술 | 쓰러지는 턴도 의미 있게 만들기 위함 |
| 교체 직후 | 첫 턴 보너스형 기술 | 후보 투입을 템포 손실로만 만들지 않기 위함 |
| 탱커 퇴장 후 | 다음 자극 몬스터 보호 장치 | 역할 교대 흐름을 만들기 위함 |
| 서포터 퇴장 후 | 감정 상태를 유지하는 잔존 효과 | 조합 플레이 보상 강화 |
| 후보 마지막 1마리 | 고위험 피니시 선택지 | 전투 막판 역전 여지 제공 |

## 퇴화/방치형 연계 명세

| 항목 | 기획 기준 |
|---|---|
| 퇴화의 의미 | 단순 페널티가 아니라 로스터 순환을 유도하는 핵심 경제 장치 |
| 전투와의 연결 | 특정 핵심 몬스터가 빠져도 팀이 굴러가도록 역할 분산 필요 |
| 스킬 배분 원칙 | 모든 필수 역할군은 최소 2종 이상 몬스터에 분산 배치 |
| 희소성 설계 | 강력한 스킬은 일부 몬스터 전용으로 두되, 필수 기능은 독점시키지 않음 |
| 복귀 보상 | 퇴화 완료 후 복귀한 몬스터는 새 스킬, 강화형 기본기, 전용 시너지 등 보상 후보 |
| 방치형 적합성 | 재접속 시 전력 복구가 일어나므로, 팀 전체에 완충 역할군이 필요 |

## 역할군별 수치 예산표

| 역할군 | power 기준 | escapeRisk 기준 | PP 기준 | 부가효과 기준 | 설계 메모 |
|---|---:|---:|---:|---|---|
| 기본 자극기 | 8~10 | 3~4 | 6~8 | 낮은 확률 `calm` 또는 `curious` | 가장 안정적인 기본 운영 기술 |
| 고위험 자극기 | 11~14 | 5~8 | 3~6 | 중간 확률 감정 유도 가능 | 빠른 순화 압박용, 남발 금지 |
| 안정화기 | 4~7 | -2~-6 | 4~6 | `calm`, 도주 억제 계열 우선 | 수치보다 실패 방지 가치가 중요 |
| 감정 유도기 | 6~9 | 2~5 | 4~7 | `curious`, `charmed`, `trust` 중심 | 직접 효율보다 후속 턴 기대값이 중요 |
| 교감 마무리기 | 14~18 | 10~16 | 2~3 | 조건부 `trust`, `charmed` 보너스 가능 | 낮은 순화 상태에서 남발 시 비효율이어야 함 |
| 수비/버티기 | 4~6 | -2~-6 | 4~6 | heal 2~9, defense 2~6 | 전투 지속과 교체 연결 역할 |
| 교체 연계기 | 5~9 | -3~4 | 3~5 | 다음 턴 보조, 감정 유지, 첫 턴 보너스 | 후보 진입을 전술화하는 기술 |

## 감정 유도 확률 가이드

| 감정 | 기본 권장 확률 | 고성능 스킬 최대 권장 | 비고 |
|---|---:|---:|---|
| `calm` | 20~35% | 45% | 가장 안정적인 제어 감정 |
| `curious` | 20~30% | 35% | 자극 연계용 핵심 상태 |
| `charmed` | 10~20% | 30% | 교감 마무리 연계용, 과도하면 위험 |
| `trust` | 5~10% | 15% | 희소성이 있어야 함 |
| `rage` | 10~15% | 20% | 플레이어 스킬에는 주로 리스크/양날검 형태 |
| `fear` | 직접 부여 지양 | 특수 상황 한정 | 주로 시스템 자동 유발 상태로 유지 권장 |

## 전투 판단 핵심 우선순위

| 우선순위 | 규칙 |
|---|---|
| 1 | 도주 위험이 높으면 순화 가속보다 도주 억제가 우선 |
| 2 | `rage` 또는 `fear` 상태면 공격적 운영보다 복구가 우선 |
| 3 | `curious`, `charmed`, `trust`가 잡히면 그 턴 또는 다음 턴에 이득을 회수해야 함 |
| 4 | 교감 마무리기는 순화율과 도주율을 같이 보고 사용해야 함 |
| 5 | 후보 진입은 손실이 아니라 전술 전환이 되도록 설계해야 함 |

## 교체/퇴화 메타에서 금지할 설계

| 금지 항목 | 이유 |
|---|---|
| 필수 안정화기를 특정 1마리에게만 독점 부여 | 퇴화/부재 시 팀이 즉시 붕괴함 |
| 모든 마무리기를 한 감각축에만 집중 | 축 메타가 단조로워지고 조합이 고정됨 |
| 감정 유도기 없이 순화량만 높은 기술 다수 배치 | 전투가 수치 누르기 게임으로 퇴화함 |
| 수비기가 지나치게 약해 교체 전 템포 확보가 불가능한 구조 | 후보 시스템 존재 가치가 약해짐 |
| 희소 스킬이 필수 역할까지 대체 불가능하게 만드는 구조 | 수집과 편성의 자유도가 사라짐 |
| 퇴화 복귀 몬스터에게만 모든 고성능 스킬 보상 | 장기 밸런스가 무너지고 신규/대체 로스터가 죽음 |

## 데이터 작성 실무 기준

| 항목 | 규칙 |
|---|---|
| 기본 원칙 | 새 스킬은 먼저 어느 역할군인지 확정한 뒤 수치를 배정 |
| 위험 관리 | `power`를 올리면 `escapeRisk`, `PP`, 조건 중 최소 하나는 같이 조정 |
| 감정 설계 | 강한 감정일수록 낮은 확률 또는 조건부 발동으로 제한 |
| 역할 분산 | 같은 역할군이라도 최소 2개 이상의 축에 분산 배치 권장 |
| 몬스터 배분 | 한 몬스터의 3스킬은 `가속 + 안정 + 마무리` 혹은 `세팅 + 유지 + 마무리` 식으로 조합 |
| 후보 가치 | 후보 몬스터는 주전 하위호환이 아니라 다른 상황을 해결하는 역할을 가져야 함 |

## 적 행동 설계 원칙

| 항목 | 기준 |
|---|---|
| 적의 전투 방식 | 적은 플레이어처럼 스킬 목록을 쓰지 않고, 성향과 상태에 따라 행동 패턴을 선택 |
| 적의 역할 | 적은 피해 딜러가 아니라 `압박 장치 + 반응형 퍼즐`로 설계 |
| 차별화 수단 | 공격 유형, 도주 성향, 감정 민감도, 타겟팅 성향, 순화 단계별 행동 변화 |
| 반복 방지 핵심 | 같은 적이라도 전투 국면과 플레이어 행동에 따라 반응이 바뀌어야 함 |
| 전투 재미 원천 | “이번 적은 어떤 반응을 보일까”를 읽는 판단 재미 |

## 적 행동 아키타입

| 아키타입 | 핵심 특징 | 주요 압박 | 플레이어 카운터 |
|---|---|---|---|
| 난폭형 | 단일 공격 강함, rage 전환 쉬움 | 특정 아군 급사 위험 | 안정화기, 수비/버티기 |
| 광역형 | 전체 공격 비중 높음 | 주전 3마리 동시 소모 | 광역 대비 수비, 후보 운영 |
| 겁쟁이형 | 도주 시도 잦음, 공격은 약함 | 순화 전에 전투 종료 | 도주 억제기, 빠른 순화 |
| 관찰형 | 공격 대신 관찰/머뭇거림이 많음 | 템포를 흔들고 판단 유도 | 감정 유도 후 마무리 |
| 집요형 | 특정 아군 집중 공격 | 핵심 몬스터 탈락 | 어그로 분산, 교체 연계 |
| 변덕형 | 감정 상태에 따라 행동이 크게 달라짐 | 예측 난이도 상승 | 감정 관리, 안전 운영 |

## 적 공격 패턴 분류

| 패턴 | 설명 | 설계 목적 |
|---|---|---|
| 단일 집중 공격 | 가장 위험한 아군 1마리 집중 | 탱커/교체 가치 강화 |
| 전체 공격 | 약한 광역 압박 | 후보 포함 장기전 압박 |
| 준비 후 강공격 | 한 턴 예고 후 강한 공격 | 방어/안정화 선택 강제 |
| 연속 약공격 | 낮은 피해를 여러 번 누적 | 회복과 교체 타이밍 흔들기 |
| 도주 행동 | 직접 피해 없이 도주 게이지 가속 | 순화/안정화 판단 갈등 유발 |
| 관찰 행동 | 공격 대신 반응/머뭇거림 | 전투 분위기와 국면 전환 |

## 순화 단계별 적 행동 변화

| 순화 구간 | 적 기본 태도 | 권장 행동 변화 |
|---|---|---|
| 0~30% | 경계, 공격적 | 직접 공격 비중 높음 |
| 30~60% | 흔들림 시작 | 공격/관찰/짧은 도주 시도 혼합 |
| 60~80% | 관심 또는 불안 | 관찰 증가, 감정 반응 확대 |
| 80%+ | 교감 직전 또는 패닉 | 공격 급감, 도주/머뭇거림/감정 폭주 중 하나 강화 |

## 감정 상태별 적 행동 수정치

| 감정 | 행동 변화 | 의도 |
|---|---|---|
| `calm` | 공격 빈도 감소, 관찰 증가 | 마무리 기회 제공 |
| `curious` | 관찰/접근 행동 증가 | 자극 연계 보상 |
| `charmed` | 공격성 감소, 교감 성공 기대값 상승 | 피니시 상태 강조 |
| `trust` | 공격 거의 중단, 도주 억제 | 안정적인 승리 구간 형성 |
| `rage` | 공격 빈도와 강도 상승, 광역 가능성 증가 | 실패 복구 구간 제공 |
| `fear` | 도주 시도 빈도 급상승 | 막판 긴장감 형성 |

## 반복 방지 규칙

| 규칙 | 목적 |
|---|---|
| 같은 감각축 반복 사용 시 적 반응 변화 | 최적 루틴 고착 방지 |
| 같은 종류의 적도 성격/도주 성향을 다르게 배정 | 전투 개성 유지 |
| 적은 턴 경과에 따라 행동 테이블 일부 변경 | 장기전 반복감 감소 |
| 예고형 행동을 섞어 유저가 대응 선택을 하게 함 | 수동적 반복 입력 방지 |
| 일부 적은 특정 감정에 강하고 다른 감정에 약하게 설계 | 감정 메타 다양화 |
| 후보 투입 직후 위협 대상이 바뀌는 적 포함 | 교체를 단순 소모가 아닌 전략으로 전환 |

## 적 설계 금지 규칙

| 금지 항목 | 이유 |
|---|---|
| 모든 적이 결국 단일 공격 + 낮은 확률 광역만 수행 | 전투 체감이 빠르게 동일해짐 |
| 도주형 적도 공격형 적도 체감 차이가 없는 수치 차이만 존재 | 적 개성이 사라짐 |
| 순화가 높아져도 적 행동이 거의 바뀌지 않음 | 전투 국면 변화 부재 |
| `rage`와 `fear`가 수치 차이만 있고 행동 변화가 없음 | 감정 시스템 존재감 상실 |
| 예고 없이 강한 피해만 주는 패턴 반복 | 불쾌감은 크고 전략성은 낮음 |
| 교체 이후에도 적이 아무 반응 변화 없이 동일 루틴 반복 | 후보 시스템의 재미 약화 |

## 아군 역할군 vs 적 아키타입 상성표

| 아군 역할군 | 강한 적 유형 | 약한 적 유형 | 이유 |
|---|---|---|---|
| 기본 자극기 | 관찰형, 겁쟁이형 | 난폭형 | 안정적 운영엔 좋지만 강한 압박엔 약함 |
| 고위험 자극기 | 관찰형, `calm` 상태 적 | 겁쟁이형, `fear` 상태 적 | 도주 가속 리스크가 큼 |
| 안정화기 | 겁쟁이형, 난폭형, `rage` 적 | 관찰형 | 안전 확보엔 강하지만 속도는 느림 |
| 감정 유도기 | 변덕형, 관찰형 | 광역형 | 세팅 가치가 있으나 광역 압박엔 늦을 수 있음 |
| 교감 마무리기 | `calm`, `trust`, `charmed` 적 | `rage`, `fear` 적 | 상태가 잡힌 적에겐 강하고 불안정 상태엔 약함 |
| 수비/버티기 | 난폭형, 광역형, 집요형 | 겁쟁이형 | 생존 확보는 좋지만 도주형에겐 느림 |
| 교체 연계기 | 집요형, 광역형 | 순수 관찰형 | 교체 템포를 전술로 바꾸는 용도 |
