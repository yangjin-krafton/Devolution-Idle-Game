# 전투 개선안 정리

> 작성일: 2026-03-13
> 목적: 현재 전투의 반복 패턴과 무력감을 줄이고, 읽히는 정보와 조합형 선택을 강화하기 위한 방향을 정리한다.

---

## 1. 현재 전투의 핵심 문제

### 1-1. 시작 패턴이 너무 고정적이다

지금 전투는 거의 항상 낮은 호감도와 낮은 도주 상태에서 시작하므로, 초반 행동이 쉽게 고정된다.

- 자극 스킬 반복
- 어느 정도 수치가 차면 포획
- 위험하면 방어

이 구조는 익히기 쉽지만, 전투마다 다른 판단을 만들지 못한다.

### 1-2. 스킬 결과를 누르기 전에 이해하기 어렵다

전투 엔진 내부에는 미리보기 계산이 이미 있지만, 유저가 실제로 이해할 수 있는 정보는 부족하다.

- 왜 이 스킬이 지금 좋은지
- 왜 이 스킬이 지금 막히는지
- 어떤 감정이나 상황을 노려야 하는지
- 이 턴의 선택이 다음 턴에 무엇을 여는지

즉, 계산은 있지만 의사결정 정보로 충분히 번역되지 않는다.

### 1-3. 스킬이 너무 즉시형, 단일 기능 중심이다

현재 스킬은 대체로 아래처럼 읽힌다.

- `stimulate`: 호감도 올리기
- `capture`: 마무리
- `defend`: 버티기

문제는 많은 스킬이 "누르면 바로 수치가 크게 바뀌는가"로만 가치가 정해진다는 점이다. 그러면 조합보다 즉시 효율만 남고, 손패가 꼬이면 할 수 있는 행동이 줄어든다.

### 1-4. 불리한 상황에서 복구 루트가 약하다

좋은 전투는 불리한 턴에도 차선책이 있어야 한다.

- 다음 턴을 준비하는 행동
- 적의 강한 행동을 흘리는 대응
- 리스크를 줄이는 완충 행동
- 약한 행동이지만 이후를 여는 연결 행동

현재 전투는 이런 중간 목표가 약해서, 플레이어가 "불리하다"보다 "할 수 있는 게 없다"로 느끼기 쉽다.

### 1-5. 적이 단순 공격 위주로 읽힌다

지금 적은 위협적일 수는 있어도, 플레이어가 대응 계획을 세울 수 있는 정보가 많지 않다. 적이 다음에 무엇을 하려는지, 그것에 어떤 대응이 좋은지 읽히는 구조가 더 필요하다.

---

## 2. 개선 목표

전투 개선의 목표는 난이도를 단순히 낮추는 것이 아니다. 매 턴의 의미를 높이고, 약한 상황에서도 계획 감각을 남기는 것이다.

핵심 목표:

1. 전투 시작부터 매번 다른 질문이 생겨야 한다.
2. 모든 스킬은 즉시 수치 외에 다음 턴 가치도 가져야 한다.
3. 모든 스킬은 예상 변화치를 읽을 수 있어야 한다.
4. 적 행동은 단순 공격이 아니라 대응 가능한 intent로 보여야 한다.
5. 몬스터마다 다른 전투 감각이 있어야 한다.

---

## 3. 설계 원칙

### 3-1. 시작 상태부터 변주를 준다

전투를 항상 같은 초기 상태에서 시작하면 플레이어도 같은 루틴으로 시작한다. 전투 시작 시점부터 상태가 다르게 열려야 한다.

### 3-2. 스킬 하나의 가치는 즉시 효과와 연결 효과를 함께 본다

강한 스킬보다 중요한 것은 죽은 턴을 줄이는 것이다.

좋은 스킬은 다음 중 둘 이상을 가져야 한다.

- 즉시 호감도/도주 변화
- 감정 세팅
- 다음 축 보너스
- 반복 패널티 우회
- 적 intent 대응
- 다음 포획 강화

### 3-3. 실패 턴도 다음 턴 가치를 남겨야 한다

포획 실패, 방어 선택, 약한 자극 같은 행동도 완전한 공턴이 되면 안 된다. 실패해도 다음 턴에 의미가 남아야 한다.

### 3-4. 정보는 숫자만이 아니라 판단 이유를 보여줘야 한다

유저는 단순히 `+8`을 보는 것이 아니라, 왜 `+8`인지 알고 싶어한다.

보여줘야 할 정보:

- 좋음 / 보통 / 나쁨
- 예상 호감도 변화
- 예상 도주 변화
- 적용 중인 보너스/패널티
- 조건 미충족 이유
- 다음 턴 연계 정보

### 3-5. 적 행동은 대응 퍼즐이어야 한다

적이 무엇을 하려는지 보이면, 플레이어 스킬의 의미도 다시 살아난다. 방어, 안정화, 셋업, 마무리의 가치가 적 intent에 따라 달라져야 한다.

## 4. 구체 개선안

### 4-1. 전투 시작 시 초기 호감도와 도주 게이지를 랜덤 부여

적 등장 시 한 번의 랜덤 롤로 초기 상태를 만든다.

필요한 방향:

- 전투 시작 시 `초기 호감도`, `초기 도주 게이지`를 랜덤 부여
- 완전 균등 랜덤이 아니라 몬스터 성격과 야생 메커닉에 맞는 범위 사용
- 일부 적은 경계 상태로 시작
- 일부 적은 호기심 상태로 시작
- 일부 적은 이미 도주 압박을 가진 상태로 시작

예시:

- 온순형: 호감도 10~20, 도주 0~10
- 경계형: 호감도 0~8, 도주 12~25
- 호기심형: 호감도 15~30, 도주 5~15

기대 효과:

- 초반 자극 반복 정답 패턴 완화
- 시작부터 포획 루트, 안정화 루트, 셋업 루트가 갈림
- 적마다 체감이 달라짐

### 4-2. 즉시 대량 증감형에서 조합형 스킬 설계로 전환

이제 모든 스킬이 즉시 큰 수치를 움직이는 구조는 줄여야 한다.

앞으로 필요한 스킬 역할:

- `setup`: 즉시 변화는 작지만 감정이나 상태를 만든다
- `bridge`: 축 전환, 감정 전환, 반복 패널티 우회
- `stabilize`: 도주 억제, 분노 정리, 방어
- `payoff`: 조건이 맞으면 큰 호감도 증가 또는 큰 도주 감소
- `finish`: 조건 충족 시 강한 포획 시도

권장 방향:

- 소폭 누적형 스킬 확대
- 특정 감정/조건을 만드는 셋업 스킬 추가
- 다음 행동을 크게 증폭시키는 트리거 스킬 추가
- 특정 적 intent에 대응하는 카운터 스킬 추가
- 한 번에 크게 터지는 기술은 조건부로 제한

예시:

- `Probe`: 호감도 +2, 다음 smell 계열 효과 +50%
- `Tune`: 즉시 변화 없음, calm이면 다음 포획 확률 +20%
- `Anchor`: 도주 -4, 다음 적 `panic` intent 약화
- `Latch`: 포획 실패 시 집중 스택 +1, 2스택 이상이면 다음 포획 강화

### 4-3. 모든 스킬에 실시간 예상 변화치 표시

이 항목은 필수다. 유저는 스킬을 누르기 전에 결과를 이해할 수 있어야 한다.

표시해야 할 항목:

- 예상 호감도 변화
- 예상 도주 변화
- 예상 감정 변화 또는 추가 효과
- 조건 미충족 이유
- 적용 중인 보너스/패널티
- 다음 턴 연계 정보

중요한 점은 계산 방식이다.

단순 현재 상태 기준 계산으로는 부족하다. 유저가 스킬을 하나씩 고를 때마다, 현재 선택 중인 다른 아군 스킬과 실행 순서를 반영해 예상값이 즉시 갱신되어야 한다.

즉:

- 1번 아군 행동을 고르면 2번, 3번 카드의 예상값도 같이 바뀐다
- 행동 순서가 바뀌면 뒤에 실행될 스킬의 예상 수치도 바뀐다
- 현재 턴의 선택 조합을 기준으로 프리뷰를 시뮬레이션해야 한다

표시 예시:

```text
좋음
예상 호감도 +4
예상 도주 +1
앞선 아군의 calm 유발 후 실행
다음 capture 확률 증가
```

즉, 프리뷰는 단순 툴팁이 아니라 "현재 선택 조합 기반 턴 예측기"가 되어야 한다.

### 4-4. 포획과 방어를 공턴이 아닌 준비 행동으로 재설계

포획 실패나 방어 선택이 완전 손해처럼 느껴지면 전투가 답답해진다.

개선 방향:

- 포획 실패 시 집중 스택, 관심 고정, 다음 포획 보정 중 하나 부여
- 일부 포획기는 누적형, 일부는 마무리형으로 역할 분리
- 방어 스킬은 회복만이 아니라 도주 억제, 감정 정리, 다음 턴 보정까지 포함
- 셋업/안정화 행동이 다음 턴의 강한 payoff로 이어지게 설계

### 4-5. 적 행동 예고를 intent 시스템으로 확장

이제 적은 단순 공격 예고만 하는 구조를 넘어서야 한다. `Slay the Spire`처럼 적 턴의 예상 행동을 다양한 intent로 보여주는 시스템이 필요하다.

권장 intent 타입:

- `attack`: 단일 공격
- `heavy_attack`: 강공
- `aoe_attack`: 전체 공격
- `guard`: 방어 강화
- `focus`: 다음 턴 강화 준비
- `observe`: 이번 턴 관찰, 다음 반응 변화
- `agitate`: 도주 게이지 증가 또는 분노 유발
- `panic`: 도주 크게 증가
- `recover`: 스스로 안정화
- `shift_axis`: 유효 축 변경

이 시스템의 목적:

- 플레이어가 적의 다음 행동을 보고 내 선택을 다시 평가하게 만들기
- 방어, 셋업, 마무리, 안정화 스킬의 가치가 상황마다 달라지게 만들기

예시:

- 적이 `heavy_attack` 예고: 방어/완화 스킬 가치 상승
- 적이 `panic` 예고: 도주 억제와 마무리 중 선택 발생
- 적이 `observe` 예고: 셋업 스킬 사용 타이밍 생성
- 적이 `guard` 예고: 직접 포획보다 감정 세팅이 유리

### 4-6. 몬스터별 행동 풀 설계 및 등록

intent 시스템은 공용 랜덤 행동만으로는 금방 단조로워진다. 각 야생 몬스터마다 고유 행동 풀을 설계해서 데이터에 등록해야 한다.

각 몬스터에 필요한 항목:

- 기본 행동 풀
- 행동 가중치
- 특정 구간 조건
- 특정 감정 상태 조건
- 야생 메커닉과 연결된 고유 행동

예시 구조:

```js
enemyBehaviorPool: [
  { id: 'bite', type: 'attack', weight: 4 },
  { id: 'watch', type: 'observe', weight: 2 },
  { id: 'spook', type: 'agitate', weight: 3, whenEscapeBelow: 60 },
  { id: 'flee_burst', type: 'panic', weight: 5, whenEscapeAbove: 70 },
]
```

기대 효과:

- 몬스터마다 다른 전투 감각 형성
- 특정 적에 맞는 대응 루트 학습
- 단순 수치가 아닌 행동 패턴으로 적을 기억하게 됨

### 4-7. 적 intent와 플레이어 프리뷰를 연결

적 행동 예고는 단독 기능으로 끝나면 안 된다. 플레이어 스킬 프리뷰와 연결될 때 가치가 커진다.

예시:

- 적 intent가 `heavy_attack`이면 방어 카드 예상 가치 강조
- 적 intent가 `panic`이면 도주 억제 스킬 예상 가치 상승 표시
- 적 intent가 `observe`이면 셋업 스킬 추천
- 적 intent가 `guard`이면 직접 포획보다 감정 세팅 추천

즉, UI는 "내 스킬이 얼마나 오르는가"만 보여주지 말고, "적 intent에 대해 왜 이게 좋은가"도 함께 설명해야 한다.

## 5. 우선순위 제안

### 1순위: 즉시 체감 개선

- 전투 시작 시 초기 호감도/도주 랜덤 부여
- 실시간 예상 변화치 계산기 구축
- 조건 미충족 사유 노출
- 보너스/패널티 가시화

### 2순위: 죽은 턴 줄이기

- 스킬 역할 재설계: 즉시형에서 조합형으로 전환
- 포획 실패 보상 추가
- 방어 스킬에 준비 기능 추가
- 브리지 스킬 확충

### 3순위: 대응 전투 확장

- 적 intent 시스템 추가
- 몬스터별 행동 풀 설계
- 적 intent와 플레이어 프리뷰 연동

## 6. 현재 스킬 테이블 기준 구체 수정안

현재 스킬 테이블은 아래 카탈로그를 중심으로 구성되어 있다.

- `src/data/skills/catalogs/stimulate/hybrid.js`
- `src/data/skills/catalogs/capture/hybrid.js`
- `src/data/skills/catalogs/defend/hybrid.js`
- `src/data/skills/catalogs/capture/advanced.js`
- `src/data/skills/catalogs/defend/advanced.js`
- `src/data/skills/catalogs/stimulate/*.js`

실제 전투 감각은 축별 개별 스킬보다 `hybrid` 템플릿 묶음이 더 크게 좌우한다. 따라서 1차 수정도 개별 스킬 이름보다 역할군 템플릿 단위로 들어가는 것이 효율적이다.

### 6-1. 자극 스킬 템플릿 수정안

현재 `stimulate/hybrid.js`에는 아래 역할이 있다.

- `cadence`: 기본 자극
- `hush`, `spiral`, `veil`, `lure`, `bloom`: 감정 세팅
- `spark`, `surge`: 고위험 자극
- `tether`, `bridge`: 교대/연계
- `sweep`: 강한 기본 자극

문제:

- 많은 스킬이 결국 "즉시 호감도 + 감정 확률" 조합으로 비슷하게 읽힌다
- `spark`, `surge`, `sweep`, `lure`는 수치 차이는 있어도 전투 감각이 크게 다르지 않다
- `tether`, `bridge`는 교대 시너지가 있지만 현재 엔진에서 교대가 핵심 루프로 살아 있지 않다

권장 수정:

| 현재 템플릿 | 현재 역할 | 수정 방향 | 구체 제안 |
|---|---|---|---|
| `cadence` | 무난한 기본 자극 | 유지 | 초반 안정형 기준점으로 유지 |
| `hush` | calm 세팅 | 강화 | `rage` 해제, 다음 defend/pacify 보너스 추가 |
| `spiral` | 감정 연계 | 유지 | "앞선 아군이 특정 감정 유발 시 추가 보너스" 명시 |
| `spark` | 고위험 큰 수치 | 축소 | 즉시 수치 감소, 대신 다음 payoff 증폭 부여 |
| `tether` | 교대용 세팅 | 재설계 | 교대가 아니라 "다음 아군 행동 +n" 보조형으로 전환 |
| `veil` | 약한 감정 세팅 | 재설계 | 도주 억제 또는 적 intent 약화 기능 추가 |
| `sweep` | 큰 기본 자극 | 축소 | 광범위 범용기 대신 특정 intent 대응형으로 분화 |
| `lure` | 강한 감정 부여 | 유지 | 포획 전 셋업용으로 역할 고정 |
| `surge` | 조건부 고위험 | 재설계 | 조건 만족 시 큰 payoff, 실패 시 다음 턴 보정 남김 |
| `bridge` | 교대 연계 | 재설계 | 반복 패널티 제거 또는 축 전환 보너스 부여 |
| `bloom` | 같은 감정 보너스 | 강화 | "현재 감정 유지 시 payoff 확대"의 핵심 셋업기로 사용 |

추천 신규 역할 매핑:

- `cadence`, `sweep` -> `setup_basic`
- `hush`, `veil` -> `stabilize_setup`
- `spiral`, `bloom`, `lure` -> `combo_setup`
- `spark`, `surge` -> `payoff_stimulate`
- `tether`, `bridge` -> `bridge`

즉, 자극 스킬은 이제 "즉시 큰 수치"가 아니라 "상황 조성" 쪽으로 더 이동해야 한다.

### 6-1-A. 자극 스킬 전용 설계 규칙

사용자 방향 기준으로, 자극 스킬은 아래 규칙을 중심으로 다시 설계한다.

#### 1. 상태 몬스터의 취향/선호에 따라 자극 강도 보너스 적용

기본 축 상성만으로는 부족하다. 앞으로는 상태 몬스터가

- 좋아하는 자극
- 싫어하는 자극
- 지금 원하는 자극

을 구분해서 자극 수치에 보너스 퍼센트를 부여하는 구조가 필요하다.

권장 구조:

- `preferredStimulus`: 항상 좋아하는 축
- `dislikedStimulus`: 항상 싫어하는 축
- `wantedStimulus`: 현재 전투 상태에서 원하는 축

적용 방식 예시:

- 좋아하는 자극: 호감도 효과 `+20% ~ +35%`
- 싫어하는 자극: 호감도 효과 `-15% ~ -30%`, 도주 리스크 추가
- 원하는 자극: 이번 턴 한정 `+40%` 수준의 강한 보너스

즉, 현재 `sensoryType` 기반 상성을 더 세분화해서 "몬스터 취향을 읽는 재미"로 확장한다.

#### 2. 강력한 자극 필살기는 지연 발동형도 넣는다

강한 자극이 전부 즉시 발동하면, 결국 제일 센 버튼을 좋은 타이밍마다 누르는 게임이 된다.

따라서 일부 강력 자극은 다음 구조를 가져야 한다.

- 사용 즉시 준비 상태 진입
- 1~2턴 뒤 발동
- 발동 전 취소되거나 약화될 수도 있음
- 발동 시 큰 payoff 발생

이 구조는 사실상 전투용 쿨타임/차징 필살기 역할이다.

기대 효과:

- 유저가 필살기 사용 시점을 전략적으로 고르게 됨
- 적 intent와 아군 셋업을 보고 미리 깔아두는 판단이 생김
- "지금 당장 효율" 대신 "2턴 뒤 payoff"가 생김

자극 템플릿 적용 예시:

- `spark` -> 즉시형 고위험에서 `charged_payoff`
- `surge` -> 조건부 즉시형에서 `delayed_finisher_stimulate`
- `chorus`, `flare`, `focus` 계열 -> 차징형 필살기로 전환

#### 3. 다른 자극을 강화/보완하는 지원형 자극 추가

자극 스킬끼리 서로 강화해야 조합 전투가 살아난다.

필요한 지원형:

- A 스킬이 B 축 자극 효과를 강화
- 특정 감정 상태에서 다른 자극 성공률을 높임
- 특정 자극의 도주 리스크를 줄여줌
- 다음 자극이 무조건 좋은 반응을 받도록 보정

예시:

- `Attune A`: 다음 `temperature` 자극은 무조건 좋아함 판정
- `Echo Link`: 다음 `sound` 자극의 호감도 배율 +50%
- `Scent Cover`: 다음 자극의 도주 증가량 -40%
- `Behavior Cue`: 적의 현재 wanted 자극을 `behavior`로 고정

즉, 자극 스킬 내부에 "버프형 서포터" 계열이 필요하다.

#### 4. 적에게 디버프를 주는 자극 타입 추가

자극은 호감도만 올리는 버튼이 아니라, 적 상태를 흔드는 수단도 되어야 한다.

권장 디버프 타입:

- 적 다음 공격 약화
- 적 `panic/agitate` intent 발생률 감소
- 적의 wanted 자극 변경
- 적의 도주 게이지 상승량 감소
- 적 특정 감정 유지시간 감소 또는 고정

예시:

- `Hush`: 적 공격 의도 약화
- `Mist`: 적의 다음 도주 증가량 감소
- `Snare Tone`: 적이 다음 턴 `observe`로 치우치게 만듦
- `Soothe Pattern`: 적 분노 발동 확률 억제

즉, 자극 계열 안에도 "디버프형 컨트롤"이 들어가야 한다.

#### 5. 특정 상황 조건에서 보너스를 받는 자극 추가

지금도 일부 조건부 스킬은 있지만, 앞으로는 전투 상황 전체를 읽는 보너스형 자극이 더 필요하다.

유효한 조건 예시:

- 이번 턴 우리 아군 모두 자극 스킬 사용
- 이번 턴 적 몬스터의 공격 대상 선정 intent 상태
- 상대 몬스터 도주 게이지 50% 이상
- 앞선 아군이 같은 축 자극 사용
- 앞선 아군이 다른 축 자극 사용
- 현재 적 감정이 `calm`, `curious`, `rage` 중 무엇인가
- 이번 턴 내가 마지막 순서로 행동

이런 조건을 받는 자극은 "판 읽기"와 "팀 조합"을 강화한다.

예시:

- `Chorus Drive`: 이번 턴 아군 모두 자극 사용 시 payoff 증가
- `Panic Catcher`: 적 도주 50% 이상이면 호감도 대폭 증가
- `Target Read`: 적이 공격 대상 선정 intent일 때 도주 억제 추가
- `Final Cue`: 이번 턴 마지막 행동이면 감정 유발 확률 증가

즉, 자극 스킬도 단순 축 상성만이 아니라 턴 문맥을 읽는 기술이 되어야 한다.

### 6-1-B. 자극 역할군 재정의

위 규칙을 반영하면, 자극 스킬 역할군은 아래처럼 재정의할 수 있다.

- `baseline_stimulate`: 취향 보정 확인용 기본기
- `preference_probe`: 좋아함/싫어함/원하는 자극 탐색
- `combo_buffer`: 다른 자극 강화
- `stim_debuffer`: 적 intent나 도주 성향 약화
- `delayed_payoff`: 1~2턴 후 발동하는 강한 자극
- `conditional_payoff`: 특정 상황에서 크게 터지는 자극
- `stabilize_stimulate`: 호감도는 낮지만 리스크 관리
- `bridge_stimulate`: 축 전환, 패널티 우회, 다음 아군 보조

이 분류를 기준으로 기존 스킬을 재배치하면, 자극 계열만으로도 훨씬 다양한 선택지가 만들어진다.

### 6-1-C. 기존 자극 스킬에 대한 구체 매핑 예시

| 현재 템플릿/스킬 | 새 역할 | 수정 제안 |
|---|---|---|
| `cadence`, `*-stimulate` | `baseline_stimulate` | 현재 축 반응과 취향 보정 확인용 기준기 |
| `hush`, `lullaby`, `herb`, `mist` | `stabilize_stimulate` | calm 유도 + 도주 상승 억제 + 적 intent 약화 |
| `spiral`, `bloom`, `lure` | `combo_buffer` | 다음 축 자극 강화 또는 특정 감정 유지 보너스 |
| `veil` | `stim_debuffer` | 적 도주 증가량 감소, 다음 공격 intent 약화 |
| `spark`, `chorus`, `flare`, `focus` | `delayed_payoff` | 차징 후 발동하는 강한 자극 필살기 |
| `surge` | `conditional_payoff` | 도주 50% 이상, 전원 자극 사용 등 조건에서만 큰 payoff |
| `tether`, `bridge`, `relay` | `bridge_stimulate` | 다음 아군 강화, 반복 패널티 무시, wanted 자극 조정 |

### 6-1-D. 데이터 스키마 확장 제안

이 설계를 실제 데이터로 담으려면 자극 스킬에 몇 가지 필드를 추가하는 것이 좋다.

권장 필드:

- `preferenceBonus`: 좋아함/싫어함/원하는 자극 보정값
- `delayedEffect`: n턴 후 발동하는 효과 정의
- `stimBuff`: 다음 자극 강화 정보
- `stimDebuff`: 적 행동/도주/감정 약화 정보
- `teamConditionBonus`: 아군 전체 행동 조건 보너스
- `intentConditionBonus`: 적 intent 조건 보너스

예시:

```js
{
  role: 'delayed_payoff',
  preferenceBonus: { liked: 0.25, disliked: -0.2, wanted: 0.4 },
  delayedEffect: { delayTurns: 2, tamingPower: 14, escapeRisk: 3 },
  intentConditionBonus: { ifEnemyIntentIn: ['target_select'], tamingPowerBonus: 4 },
}
```

### 6-2. 포획 스킬 템플릿 수정안

현재 `capture/hybrid.js`에는 아래 역할이 있다.

- `snare`: 감정 보너스가 있는 기본 피니셔
- `clasp`: 순화 60% 이상 조건부 피니셔
- `pact`: 순화 70% 이상, 도주 제한 조건부 피니셔
- `sealburst`: 고위험 강한 피니셔
- `keystone`: 비교적 안전한 조건부 피니셔

문제:

- 대부분의 포획 스킬이 결국 "조건 맞으면 누르는 마무리 버튼"으로 수렴한다
- 실패 시 다음 턴 가치가 거의 남지 않는다
- 포획 스킬 간 플레이 감각 차이가 조건과 수치 차이 수준에 머문다

권장 수정:

| 현재 템플릿 | 현재 역할 | 수정 방향 | 구체 제안 |
|---|---|---|---|
| `snare` | 기본 피니셔 | 재설계 | 누적형 포획기로 전환, 실패 시 `focusCapture +1` |
| `clasp` | 60% 조건부 피니셔 | 유지 | 중간 마무리 기준점으로 유지 |
| `pact` | 고조건 피니셔 | 재설계 | 성공률은 낮추고 성공 시 대형 보상, 실패 시 감정 유지 |
| `sealburst` | 올인 피니셔 | 유지 | 리스크 극대화 버튼으로 유지 |
| `keystone` | 안전형 피니셔 | 강화 | 실패 시 도주 증가 경감 또는 집중 스택 제공 |

추가로 필요한 신규 포획 역할:

- `probe_capture`: 초반에 던지는 누적형 포획
- `lock_capture`: 적 `panic` intent를 묶는 억제형 포획
- `payoff_capture`: 특정 감정/스택에서 폭발하는 포획

`capture/advanced.js`의 고급 포획기들도 같은 방향으로 정리한다.

| 현재 스킬 | 문제 | 수정안 |
|---|---|---|
| `sound-finale` | 그냥 더 강한 고위험 피니셔로 읽힘 | 성공 시 보상 유지, 실패 시 `trust seed` 부여 |
| `smell-seal` | 호기심 보정 외에는 플레이 감각이 약함 | `observe` intent 적에게 추가 효율 부여 |
| `temperature-embrace` | 안전형 피니셔에 가까움 | `panic` 억제 보정 추가 |
| `behavior-vow` | 너무 늦게 열리는 강한 한방 | 발동 전 준비 스택형으로 분산 |

### 6-3. 방어 스킬 템플릿 수정안

현재 `defend/hybrid.js`에는 아래 역할이 있다.

- `ward`: 팀 보호
- `shelter`: 안정화
- `mend`: 단일 회복
- `pivot`: 교대 보조
- `reserve`: 방어 특화 안정화

문제:

- 지금 방어는 대부분 "회복 + 방어 + 도주 감소"에 머문다
- 생존은 되지만 다음 턴 설계로 이어지는 손맛이 약하다
- `pivot`의 교대 시너지는 현 루프에서 체감이 작다

권장 수정:

| 현재 템플릿 | 현재 역할 | 수정 방향 | 구체 제안 |
|---|---|---|---|
| `ward` | 팀 보호 | 유지 | 적 `attack/heavy_attack` intent 대응 강조 |
| `shelter` | 안정화 | 강화 | 분노 해제, 다음 자극 안정성 증가 추가 |
| `mend` | 단일 회복 | 재설계 | 개인 생존 외에 다음 포획/자극 보너스 부여 |
| `pivot` | 교대 보조 | 재설계 | "다음 아군 우선도 +1 / payoff +n" 보조형으로 전환 |
| `reserve` | 단단한 안정화 | 강화 | 반복 패널티 1회 무시 또는 도주 급증 방지 추가 |

`defend/advanced.js`의 개별 스킬도 정리 대상이다.

| 현재 스킬 | 문제 | 수정안 |
|---|---|---|
| `sound-guard` | 수치형 보호기 | `heavy_attack` intent 카운터로 명확화 |
| `temperature-heal` | 큰 회복기지만 전투 전진성 낮음 | 다음 `temperature` payoff 강화 |
| `smell-aroma-wall` | 수치 조합형 보호기 | `agitate/panic` 억제 부여 |
| `behavior-guard` | 방어 수치 위주 | 적 `focus` intent 약화 기능 추가 |
| `sound-recover` | 그냥 회복기 | calm 유지 또는 연장 부여 |
| `smell-sanctuary` | 안정화 특화 | 초반 랜덤 도주 시작 적 상대로 핵심 카운터화 |
| `temperature-anchor` | 교대 보너스 의존 | intent 지연 또는 행동 약화형으로 재설계 |

### 6-4. 축별 개별 자극 스킬 수정 원칙

개별 축 파일에도 유사 문제가 반복된다.

- `src/data/skills/catalogs/stimulate/sound.js`
- `src/data/skills/catalogs/stimulate/temperature.js`
- `src/data/skills/catalogs/stimulate/smell.js`
- `src/data/skills/catalogs/stimulate/behavior.js`

현재 경향:

- 기본형: `*-stimulate`
- 감정형: `*-lullaby`, `*-mist`, `*-herb`, `*-bow`
- 고위험형: `*-chorus`, `*-flare`, `*-sweet`, `*-focus`
- 교대형: `*-relay` 계열

구체 수정 원칙:

- 기본형은 초반 랜덤 시작 상태를 읽는 기준기 역할로 유지
- 감정형은 감정 하나만 거는 기술이 아니라, 특정 intent 대응 정보까지 가진다
- 고위험형은 즉시 수치를 깎고 payoff 증폭기로 전환한다
- 교대형은 실제 교대가 약하면 "다음 아군 강화" 역할로 치환한다

예시:

| 현재 스킬 | 수정 방향 |
|---|---|
| `sound-stimulate` | 초반 기준 자극기로 유지 |
| `sound-lullaby` | `rage` 해제 + 다음 포획 소폭 보정 |
| `sound-chorus` | 즉시 큰 수치 대신 2턴형 payoff 준비기로 전환 |
| `temperature-flare` | `panic` 직전 적 상대 위험 감수 payoff로 전환 |
| `smell-herb` | 안정 calm/charmed 셋업기로 유지 |
| `smell-sweet` | 단순 고위험이 아니라 `observe` intent 카운터로 조정 |
| `behavior-focus` | 큰 한방 자극에서 "다음 behavior payoff 증폭"으로 전환 |
| `behavior-relay` | 교대 대신 아군 순서/우선도 보조로 전환 |

### 6-5. 실제 1차 수정 범위 제안

전체 123개를 한 번에 갈아엎지 말고, 템플릿 기반으로 1차 수정 범위를 제한한다.

1차 수정 추천 범위:

1. `stimulate/hybrid.js`
2. `capture/hybrid.js`
3. `defend/hybrid.js`
4. `capture/advanced.js`
5. `defend/advanced.js`

이렇게 하면 대부분의 확장 스킬 감각을 한 번에 바꿀 수 있다.

1차 수정 목표:

- 자극: 30%는 기준기, 40%는 셋업/브리지, 30%는 payoff
- 포획: 40%는 누적형/중간형, 40%는 조건부 마무리, 20%는 올인형
- 방어: 50%는 안정화, 30%는 다음 턴 보조, 20%는 강한 카운터형

### 6-6. 문서 기준 최종 판단

현재 스킬 테이블은 양은 충분하다. 문제는 수보다 역할 분화 방식이다.

정리하면:

- 자극 스킬은 즉시 효율에서 조합 준비 쪽으로 이동
- 포획 스킬은 전부 마무리 버튼이 아니라 누적형과 락형을 추가
- 방어 스킬은 생존기에서 대응기/보조기로 확장
- 교대 시너지 의존 템플릿은 현재 루프에 맞게 "다음 아군 강화"로 치환

즉, 새 스킬을 더 많이 추가하는 것보다 현재 역할군을 재배치하는 것이 우선이다.

## 7. 구현 시 체크 포인트

- 랜덤 시작 수치가 너무 커서 전투 공정성을 해치지 않는가
- 실시간 프리뷰 계산이 복잡해져도 UI 응답성이 유지되는가
- 조합형 스킬이 많아져도 유저가 2턴 안에 이해 가능한가
- 포획 실패 보상이 과해서 긴장감이 죽지 않는가
- 적 intent가 다양해져도 몬스터 개성이 흐려지지 않는가

## 8. 결론

현재 전투의 문제는 단순 밸런스 수치보다 구조의 문제에 가깝다.

핵심은 다음 네 가지다.

- 전투 시작 상태에 변주를 준다
- 스킬을 즉시형에서 조합형으로 바꾼다
- 모든 스킬의 예상 변화를 실시간으로 읽히게 만든다
- 적을 단순 공격자가 아니라 intent를 가진 대응 대상으로 만든다

이 방향으로 가면 전투는 "자극 반복 후 포획" 루프에서 벗어나, 상황을 읽고 흐름을 설계하는 전투로 바뀔 수 있다.
