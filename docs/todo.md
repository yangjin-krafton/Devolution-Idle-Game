## Phase 3: 스킬 3분류 체계

현재: 순화 행동 / 인간 교감 (2탭 분리)
목표: 자극 / 포획 / 수비 (3계열, 몬스터 스킬에 통합)

### 3-1. 데이터 구조 변경

- [x] **스킬에 `category` 필드 추가**
  - 파일: `data.js` → 각 아군 몬스터의 `actions[]`
  - `category`: `'stimulate'` | `'capture'` | `'defend'`
  - 기존 순화 행동 → 자극 계열로 매핑
  - 인간 교감 행동 → 포획 계열 스킬로 몬스터에 통합
  - 수비 계열 스킬 신규 추가

- [x] **`BONDING_ACTIONS` 제거, 몬스터 스킬로 통합**
  - 포획 계열 스킬을 각 몬스터의 `actions[]`에 포함
  - 기존 `useBonding()` 로직 → `useAction()`에서 category 분기
  - `actionPanelUI.js`의 2탭 구조 제거

### 3-2. 계열별 효과 구현

- [x] **자극 계열 효과**
  - 파일: `combat.js`
  - 적 순화 게이지 ↑ (기존 로직과 유사)
  - 감각축 선호도에 따른 효과 차이 유지

- [x] **포획 계열 효과**
  - 파일: `combat.js`
  - 순화도 기반 성공 확률 계산 (기존 `useBonding` 로직 활용)
  - 성공 → 전투 승리
  - 실패 → 도주 게이지 급상승

- [x] **수비 계열 효과**
  - 파일: `combat.js`
  - 방어: 다음 적 공격 피해 감소 or 무효화
  - 회복: 아군 HP 회복
  - 약화: 적 공격력 임시 감소
  - 부작용: 순화 게이지 소폭 감소 + 도주 게이지도 감소

### 3-3. UI 변경

- [x] **액션 패널 단일 탭으로 통합**
  - 파일: `ui/actionPanelUI.js`
  - 2탭(순화/교감) 구조 제거
  - 스킬 카드에 계열 아이콘/색상 표시
    - 자극: 기존 감각축 색상
    - 포획: `C.orange`
    - 수비: 새 색상 (파란계열 등)
  - 포획 스킬: 순화도 부족 시 경고 표시 (사용은 가능하되 리스크 표시)

## Phase 4: 스탯 시스템

현재: `power`, `escapeRisk` 고정값
목표: 몬스터 스탯 × 스킬 위력% 계산식

### 4-1. 몬스터 스탯 정의

- [x] **아군 몬스터에 스탯 추가**
  - 파일: `data.js`
  - 스탯 종류 (MVP): `gentleness`(순화력), `empathy`(교감력), `resilience`(방어력), `agility`(회피/선제)
  - 야생 상태: 4개 스탯 고르게 분산
  - 퇴화 후: 1~2개에 집중 → 역할 특화

- [x] **적 몬스터에 전투 스탯 추가**
  - 현재 `attackPower` 만 있음 → 적 스탯 체계 추가
  - 적 공격력, 방어력 등

### 4-2. 계산식 구현

- [x] **스킬 효과 = 스탯 × 스킬 위력%**
  - 파일: `combat.js`
  - 자극 스킬: `순화력 스탯 × 스킬위력% × 감각축 선호도`
  - 포획 스킬: `교감력 스탯 × 스킬위력%` 기반 성공 확률
  - 수비 스킬: `방어력 스탯 × 스킬위력%` 기반 방어/회복량
  - 적 공격: `적 공격력 스탯` 기반 피해

### 4-3. 퇴화 시 스탯 변화

- [x] **퇴화 후 스탯 재분배**
  - 파일: `team.js` → 퇴화 처리 로직
  - 총 스탯 합계는 유지하되 1~2개에 집중
  - 예: 야생(5/5/5/5) → 퇴화 후(12/3/3/2) — 순화력 특화

## 구현 순서 권장

```
Phase 1 (게이지 바) → Phase 3 (스킬 3분류) → Phase 2 (3v1 턴) → Phase 4 (스탯) → Phase 5 (피드백)
```

Phase 1은 UI만 바꾸면 되서 가장 빠르고, 즉시 플레이 경험 개선.
Phase 3은 현재 2탭 구조를 통합하는 것이라 Phase 2보다 선행하는 게 자연스러움.
Phase 2는 전투 루프 전체 개편이라 가장 큰 작업.
Phase 4, 5는 핵심 루프 완성 후 밸런싱 단계.
