# 전투 시스템 환경 조율형 리뉴얼 제안

> 작성일: 2026-03-13
> 목적: 현재의 퍼센트/포획 중심 전투를, `야생 몬스터가 좋아하는 환경을 도주 전에 맞춰 주는 전투`로 재설계한다.

---

## 1. 핵심 콘셉트

이제 전투의 메인 목표는 적을 때리거나 확률을 굴리는 것이 아니다.

- 야생 몬스터는 각자 좋아하는 서식 환경이 있다.
- 플레이어는 전투 필드의 환경을 그 몬스터 취향에 맞게 조율해야 한다.
- 몬스터는 턴이 지날수록 계속 도주하려고 한다.
- 적 몬스터도 매턴 자기 환경 스킬을 랜덤하게 써서 전장을 다시 바꾼다.
- 도주 게이지가 가득 차기 전에, 몬스터가 좋아하는 환경을 `연속 3턴` 유지하면 교감/포획 성공이다.

즉 전투는 `공격전`이 아니라 `환경 세팅 퍼즐 + 도주 관리`가 된다.

스플랜더에서 가져올 핵심은 토큰 자체보다 아래 3가지다.

1. 목표가 공개되어 있다.
2. 지금 무엇을 해야 하는지 한눈에 읽힌다.
3. 몇 턴 안에 빠르게 조건을 맞추는 압축된 의사결정이 나온다.

---

## 2. 승패 구조

전투는 아래 흐름으로 정리한다.

1. 야생 몬스터가 랜덤한 환경 상태의 전장에 등장한다.
2. 몬스터의 `선호 환경 테이블`이 공개된다.
3. 플레이어는 출전 중인 아군 3명의 스킬과 후보 3명의 교체를 활용해 환경을 조절한다.
4. 적 몬스터는 자기 턴마다 환경 변화 스킬 1개를 랜덤하게 사용한다.
5. 적 턴이 지나갈 때마다 도주 게이지가 상승한다.
6. 도주 게이지가 가득 차면 전투는 실패하며, 야생 몬스터는 도주한다.
7. 5개 환경 축이 모두 적의 허용 범위 안에 들어온 상태를 `연속 3턴` 유지하면 교감 성공으로 전투 종료한다.

핵심은 간단하다.

- 기존: 수치 누적 후 확률 시도
- 신규: 환경 5축을 맞춘 뒤, 적의 방해를 버티며 3턴 유지

---

## 3. 환경 5축 정의

각 몬스터는 아래 5개 축에 대해 선호 기준을 가진다.

| 축 | 범위 | 설명 |
|------|------|------|
| 온도 | 차가움 ↔ 뜨거움 | 공기와 체감 열기 |
| 밝기 | 어두움 ↔ 밝음 | 하늘, 조명, 그림자 |
| 냄새 | 악취 ↔ 꽃향 | 향기, 독기, 풀냄새 |
| 습도 | 건조 ↔ 습함 | 안개, 물기, 공기 습도 |
| 소리 | 조용함 ↔ 소음 | 울림, 바람, 진동, 소란 |

권장 표현은 `-2 ~ +2`의 5단계 정수 트랙이다.

| 값 | 의미 |
|------|------|
| -2 | 매우 낮음 |
| -1 | 낮음 |
| 0 | 중립 |
| +1 | 높음 |
| +2 | 매우 높음 |

예시:

- 온도 `-2`: 매우 차가움
- 밝기 `+2`: 매우 밝음
- 냄새 `+2`: 꽃향이 강함
- 습도 `-2`: 매우 건조함
- 소리 `-1`: 조용한 편

이 방식이 좋은 이유는, 전투 UI와 로그에 그대로 노출하기 쉽기 때문이다.

- `온도 +1`
- `밝기 -2`
- `소리 0으로 조정`

이 정도 문법이면 플레이어가 계산 없이 즉시 이해할 수 있다.

---

## 4. 몬스터 선호 환경 테이블

각 야생 몬스터는 전투용 기준 테이블을 가진다.

권장 데이터 구조:

- `ideal`: 가장 좋아하는 목표값
- `tolerance`: 성공으로 인정하는 허용 범위
- `fleeBias`: 기본 도주 성향
- `traits`: 특정 축에 대한 추가 반응 규칙
- `environmentSkills`: 적이 자기 턴에 사용하는 환경 변화 스킬 목록

예시:

| 항목 | 값 |
|------|------|
| 몬스터 | 안개 해파리 |
| 온도 | `-1` |
| 밝기 | `-2 ~ -1` |
| 냄새 | `+1` |
| 습도 | `+2` |
| 소리 | `-1 ~ 0` |
| 기본 도주 상승 | 턴 종료 시 `+1` |
| 특수 반응 | 밝기 `+1` 이상이면 도주 추가 `+1` |

이 테이블을 전투 시작 시 적 카드에 보여주면, 플레이어는 첫 턴부터 아래를 판단할 수 있다.

- 지금 무엇이 부족한가
- 먼저 어느 축을 맞출 것인가
- 도주 게이지를 지금 막아야 하는가
- 교체해서 더 맞는 조율 스킬을 가져와야 하는가
- 적이 다음 턴 환경을 흔들어도 유지할 수 있는가

또한 적이 좋아하는 환경으로 전투가 처음부터 완성된 채 시작하는 일을 막기 위해,
전투 시작 환경은 아래 조건을 따르는 편이 좋다.

- 시작 환경은 완전 랜덤이 아니라 `최소 2개 축 이상은 어긋난 상태`로 보정
- 이미 5축이 모두 맞는 경우 재굴림
- 적은 자기 턴마다 환경 스킬을 랜덤 사용해, 맞춘 값을 다시 흔든다

---

## 5. 전장 시스템: 하늘 + 그리드를 그대로 활용

이미 구현된 `하늘 + 그리드` 전장을 환경 조율 시스템의 중심으로 사용한다.

### 5-1. 하늘 레이어가 맡을 것

하늘은 전장 전체에 영향을 주는 전역 환경으로 묶는다.

- 밝기 변화
- 온도 변화
- 습도 변화
- 날씨/구름/안개 같은 지속 효과

예시:

- `먹구름 전개`: 밝기 `-1`, 습도 `+1`
- `강한 햇살`: 밝기 `+2`, 온도 `+1`
- `안개막`: 밝기 `-1`, 습도 `+2`, 소리 `-1`

### 5-2. 그리드 레이어가 맡을 것

그리드는 위치성 있는 환경 장치를 담당한다.

- 냄새 오브젝트 배치
- 소리 발생 지점 생성
- 물웅덩이, 꽃밭, 진흙, 잔해 같은 지형 변화
- 특정 칸 주변의 국소 환경 변화

예시:

- `꽃가루 타일`: 냄새 `+1`
- `진흙 타일`: 습도 `+1`, 소리 `-1`
- `메아리 기둥`: 소리 `+2`
- `악취 균열`: 냄새 `-2`

### 5-3. 설계 원칙

- 환경 변화는 반드시 단계적으로 보인다.
- 스킬 사용 결과가 하늘과 그리드에 시각적으로 남아야 한다.
- 전투 로그와 수치는 같은 정보를 가리켜야 한다.
- 숨은 배율 대신 `현재 값`과 `목표 값`만 보이게 한다.

즉 플레이어는 "왜 값이 바뀌었는지"를 전장만 봐도 이해해야 한다.

---

## 6. 도주 게이지 규칙

야생 몬스터는 가만히 기다리지 않는다.
따라서 전투 긴장감은 HP가 아니라 `도주 게이지`가 담당한다.

기본 규칙:

- 적은 자기 턴마다 환경 변화 스킬 1개를 랜덤하게 사용한다.
- 적 턴 종료 시 도주 게이지가 기본적으로 상승한다.
- 현재 환경이 선호 범위와 많이 어긋날수록 추가 상승한다.
- 몇 개 축이 이미 맞아 있을수록 상승량이 줄어들 수 있다.
- 일부 스킬은 도주 게이지를 직접 낮추거나, 이번 턴 상승을 막는다.

권장 판정:

- 적 행동: 자기 스킬 풀에서 1개 랜덤 사용
- 기본 상승: 적 턴 종료 시 `+1`
- 크게 어긋난 축이 2개 이상: 추가 `+1`
- 5축 중 4개 이상이 허용 범위: 이번 턴 추가 상승 없음
- 5축 모두 충족: `안정 턴 수 +1`
- 한 축이라도 벗어나면: `안정 턴 수`를 0으로 초기화
- `안정 턴 수 3`: 교감 성공

즉 플레이어는 매 턴 두 가지를 동시에 봐야 한다.

- 환경을 목표에 얼마나 가깝게 만들었는가
- 그 과정에서 도주 게이지를 감당 가능한 수준으로 유지했는가

그리고 실제 플레이 감각은 아래에 더 가깝다.

- 이번 턴에 목표 환경을 만들 수 있는가
- 적이 다음 턴 랜덤 스킬을 써도 무너지지 않게 유지할 수 있는가
- 3턴 연속 유지 카운트를 지킬 수 있는가

---

## 7. 아군 스킬 구조 리뉴얼

아군 스킬은 더 이상 `자극 / 방어 / 교감` 3분류로 유지하지 않는다.
새 전투에서는 스킬이 `환경을 어떻게 바꾸는가`를 기준으로 읽혀야 한다.

권장 역할군은 아래 4가지다.

### 7-1. 조성 스킬

특정 환경 축을 직접 움직인다.

예시:

- `따스한 숨결`: 온도 `+2`
- `그늘 펼치기`: 밝기 `-2`
- `꽃가루 흩뿌리기`: 냄새 `+1`, 습도 `+1`

### 7-2. 유지 스킬

이미 맞춘 축이 흔들리지 않도록 고정하거나, 도주를 억제한다.

예시:

- `안개 고정`: 습도 변화 잠금 1턴
- `정적 유지`: 소리 변화 방지, 도주 상승 `-1`
- `은신 둥지`: 밝기와 소리를 1턴 동안 유지

이 역할군은 특히 중요하다.
이제 승리는 목표값을 한 번 찍는 것이 아니라, 적의 랜덤 환경 교란을 버티며 `3턴 연속 유지`하는 것이기 때문이다.

### 7-3. 전장 설치 스킬

하늘 또는 그리드에 지속 오브젝트를 남긴다.

예시:

- `비구름 호출`: 2턴 동안 밝기 `-1`, 습도 `+1`
- `향초 배치`: 주변 칸 냄새 `+2`
- `메아리 돌 세우기`: 해당 구역 소리 `+1`

### 7-4. 교체 연계 스킬

후보 3명과의 교체를 전투 핵심으로 만든다.

예시:

- 교체 출전 시 온도 `-1`
- 퇴장 시 향기 장판 남김
- 특정 조합 교체 시 도주 게이지 `-1`

이 구조로 가면 각 아군은 "딜러"가 아니라 아래처럼 읽힌다.

- 온도 조절 담당
- 밝기/습도 전환 담당
- 냄새 장판 설치 담당
- 소리 억제 담당
- 도주 억제 담당
- 교체 콤보 담당

---

## 8. 3인 출전 + 후보 3인 교체 구조

이번 리뉴얼에서는 `출전 3명 + 후보 3명`을 전투 시스템 핵심으로 올린다.

이 구조가 필요한 이유는, 5개 환경 축을 한 파티가 한 번에 모두 다루기 어렵기 때문이다.

권장 규칙:

- 출전 중인 3명만 즉시 행동 가능
- 후보 3명은 교체로 진입
- 교체는 강한 환경 전환 수단이지만, 턴 자원 또는 쿨다운을 가진다
- 특정 몬스터는 특정 후보를 불러와야만 빠르게 맞출 수 있다

예시 역할 배치:

- 1번 아군: 온도/밝기 조절
- 2번 아군: 냄새/습도 조절
- 3번 아군: 소리/도주 억제
- 후보 A: 극단적 어두움 세팅
- 후보 B: 꽃향 + 고습 세팅
- 후보 C: 긴급 도주 억제 및 값 고정

이렇게 하면 파티 편성 자체가 "어떤 몬스터 환경을 얼마나 빨리 맞출 수 있는가"의 문제로 바뀐다.

## 9. 스플랜더식으로 유지할 부분

이번 리뉴얼은 스플랜더를 그대로 복제하는 방향은 아니다.
대신 아래 감각은 유지하는 편이 맞다.

### 9-1. 공개 목표

적이 원하는 값이 보여야 한다.
플레이어는 숨은 배율이 아니라 목표 표를 보고 행동해야 한다.

### 9-2. 짧은 액션 문법

스킬 문장은 짧아야 한다.

- `온도 +2`
- `밝기 -1, 습도 +1`
- `소리 고정 1턴`
- `도주 게이지 -1`

이 정도로 끝나야 한 턴 판단 속도가 유지된다.

### 9-3. 빠른 누적 달성감

매 턴마다 5축 중 몇 개가 맞았는지가 보여야 한다.
즉 플레이어는 퍼센트가 아니라 `완성된 축 수`를 보면서 전진감을 느낀다.

여기에 더해, 지금 몇 턴 연속 유지 중인지도 반드시 보여야 한다.

- `안정 1/3`
- `안정 2/3`
- `안정 3/3 -> 성공`

---

## 10. 전투 예시

아래는 새 시스템 기준의 한 전투 예시다.

### 10-1. 적 목표

`황혼 박쥐`

- 온도 `-1`
- 밝기 `-2`
- 냄새 `0`
- 습도 `+1`
- 소리 `-1`

### 10-2. 시작 환경

- 온도 `+1`
- 밝기 `+1`
- 냄새 `-1`
- 습도 `-1`
- 소리 `+2`

즉 시작 시점에는 5축 모두 어긋난 상태다.

### 10-3. 플레이 흐름

1턴:

- 아군 A `먹구름 전개`: 밝기 `-1`, 습도 `+1`
- 아군 B `냉기 숨결`: 온도 `-2`
- 아군 C `정적 유지`: 소리 `-1`, 도주 상승 `-1`

결과:

- 온도 `-1` 달성
- 밝기 `0`
- 냄새 `-1`
- 습도 `0`
- 소리 `+1`
- 아직 5축 미충족, 안정 턴 수 `0/3`

2턴:

- 후보 교체 진입 효과로 냄새 `+1`
- 그리드에 `꽃이끼 타일` 설치: 냄새 `+1`, 습도 `+1`
- `음파 차단막`: 소리 `-2`
- 적 행동 `돌풍 비명`: 소리 `+1`, 밝기 `+1`

결과:

- 온도 `-1`
- 밝기 `+1`
- 냄새 `+1`
- 습도 `+1`
- 소리 `0`
- 적이 다시 환경을 흔들었기 때문에 아직 안정 턴 수 `0/3`

3턴:

- `황혼 장막`: 밝기 `-2`
- `정적 유지`: 소리 `-1`
- `중화 포자`: 냄새 `0`

결과:

- 5축 모두 충족
- 안정 턴 수 `1/3`

4턴:

- 적 행동 `습한 날갯짓`: 습도 `+1`
- 플레이어가 `건조 숨결`로 습도 `-1`
- 다른 축 유지 성공
- 안정 턴 수 `2/3`

5턴:

- 적 행동 `먹구름 펄럭임`: 밝기 `-1`, 소리 `+1`
- 플레이어가 `음파 차단막`과 `은신 둥지`로 복구
- 5축 모두 다시 유지
- 안정 턴 수 `3/3`
- 교감 성공, 전투 종료

이 흐름에서는 숫자가 많아도 읽는 방식이 매우 단순하다.

- 지금 맞은 축
- 아직 부족한 축
- 적이 이번 턴 무엇을 흔들었는지
- 안정 유지가 몇 턴째인지
- 도주가 얼마나 위험한지

이 다섯 가지만 보면 된다.

## 11. 데이터와 UI에 필요한 변경

### 11-1. 몬스터 데이터

각 몬스터에 아래 필드가 필요하다.

```js
environmentPreference: {
  temperature: { ideal: -1, tolerance: 0 },
  brightness: { ideal: -2, tolerance: 0 },
  smell: { ideal: 1, tolerance: 1 },
  humidity: { ideal: 2, tolerance: 0 },
  sound: { ideal: -1, tolerance: 1 },
},
fleeProfile: {
  baseGain: 1,
  mismatchBonus: 1,
},
environmentSkills: [
  { id: 'bat_darken', weight: 3 },
  { id: 'bat_screech', weight: 2 },
  { id: 'bat_wing_mist', weight: 1 },
],
captureRule: {
  sustainTurns: 3,
}
```

### 11-2. 전투 상태

전투 중에는 아래 상태를 추적해야 한다.

```js
battlefieldEnvironment: {
  temperature: 0,
  brightness: 1,
  smell: -1,
  humidity: 0,
  sound: 2,
},
environmentLocks: {
  temperature: 0,
  brightness: 1,
  smell: 0,
  humidity: 0,
  sound: 0,
},
escapeGauge: 3,
preferredEnvironmentStreak: 0,
lastEnemyEnvironmentSkillId: 'bat_screech',
activeFieldEffects: []
```

### 11-3. 전투 UI

전투 화면에는 아래 정보가 항상 보여야 한다.

- 현재 환경 5축
- 적의 목표 환경 5축
- 각 축의 충족 여부
- 현재 안정 유지 턴 수 `0/3`, `1/3`, `2/3`, `3/3`
- 도주 게이지
- 적이 방금 사용한 환경 스킬
- 출전 3명 / 후보 3명 상태
- 하늘과 그리드의 시각 변화

즉 상단 UI는 `적 체력바`보다 `환경 대시보드`에 가까워진다.

## 12. 구현 우선순위

### 1단계

- 환경 5축 상태와 적 목표값을 전투 UI에 표시
- 도주 게이지를 메인 실패 조건으로 승격
- 전투 시작 시 랜덤 환경 생성
- 시작 환경이 적 선호 상태로 완성되지 않도록 보정

### 2단계

- 스킬을 환경 변화형 문법으로 재작성
- 하늘 레이어와 환경값 연동
- 그리드 오브젝트와 환경값 연동
- 적 몬스터의 랜덤 환경 스킬 시스템 추가

### 3단계

- 후보 3명 교체 효과 추가
- 축 고정, 지속 장판, 전역 날씨 효과 추가
- 몬스터별 환경 테이블 데이터 정리
- `연속 3턴 유지` 승리 판정 추가

### 4단계

- 몬스터 개성 규칙 추가
- 특정 축 과잉 반응, 역치 반응, 도주 패턴 차별화
- 바이옴/시간대에 따른 랜덤 시작 환경 확장

## 13. 한 줄 결론

이번 리뉴얼의 핵심은 `야생 몬스터의 취향을 읽고, 적의 환경 교란을 버티면서 3턴 동안 그 취향을 유지하는 것`이다.

이 방향으로 가면 전투는 더 이상 `자극 수치 누적 후 포획 확률 체크`가 아니라,
`환경 5축을 만들고, 흔들리는 전장을 다시 붙잡으며 유지하는 짧고 읽기 쉬운 퍼즐`이 된다.

그리고 이미 있는 `하늘 + 그리드`, `3인 전투`, `후보 교체` 구조를 전부 자연스럽게 살릴 수 있다.
