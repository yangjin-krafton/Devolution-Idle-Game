# 전투 시스템 스플랜더형 토큰 리뉴얼 제안

> 작성일: 2026-03-13
> 목적: 현재 퍼센트/배율 중심 전투를, `스플랜더`처럼 공개 정보와 토큰 경제 중심의 읽기 쉬운 전투로 재설계한다.

---

## 1. 이 방향이 왜 맞는가

스플랜더가 좋은 이유는 복잡한 수학이 아니라, 아래 5가지가 매우 잘 읽히기 때문이다.

1. 무엇을 모아야 하는지 공개되어 있다.
2. 지금 할 수 있는 행동이 몇 가지로 압축된다.
3. 토큰은 임시 자원, 카드는 영구 할인이라 역할이 명확하다.
4. 조커 자원은 위기 탈출용이라 직관적이다.
5. 목표가 "점수 퍼센트"가 아니라 "필요 비용 충족"이라 판단이 빠르다.

지금 전투가 복잡하게 느껴지는 이유는 반대다.

- 목표치가 퍼센트다.
- 실제 효과가 배율 계산 뒤에 나온다.
- 포획은 가장 중요한 행동인데 확률 계산으로 끝난다.
- 상성, 감정, 스탯, 반복 패널티가 모두 별도 곱셈이다.

즉 현재 게임은 전투 테마는 직관적인데, 계산 언어는 직관적이지 않다.

스플랜더식으로 바꾸면 플레이어 사고는 아래처럼 단순해진다.

- 지금 적이 원하는 감각은 무엇인가
- 이번 턴에 어떤 토큰을 모을까
- 어느 할인 엔진을 먼저 깔까
- 언제 조커를 아껴서 최종 교감 비용을 맞출까

이건 "몇 퍼센트인지 계산"이 아니라, **"무엇을 모아서 언제 낼지 판단"**하는 구조다.

---

## 2. 핵심 한 줄 요약

현재의 `순화 % / 도주 % / 포획 확률` 구조를 버리고,
전투를 아래 세 층으로 바꾼다.

1. `감각 토큰`을 모은다
2. `공감 할인`을 쌓는다
3. `교감 비용`을 지불하면 성공한다

즉 전투는 데미지 레이스가 아니라, **토큰 엔진을 짧게 돌려 적의 교감 비용을 맞추는 퍼즐**이 된다.

---

## 3. 스플랜더 문법을 전투에 대응시키기

| 스플랜더 요소 | 전투 대응 |
|------|------|
| 보석 토큰 | 감각 토큰 |
| 골드 조커 | 공감 토큰 |
| 개발 카드 | 교감 카드 / 안정 카드 / 관찰 카드 |
| 영구 보너스 | 공감 할인 아이콘 |
| 귀족 타일 | 교감 마일스톤 |
| 카드 구매 비용 | 몬스터 교감 비용 |

이 대응이 중요한 이유는, 원래 복잡했던 전투 계산을 모두 "비용과 할인" 언어로 번역할 수 있기 때문이다.

---

## 4. 기본 전장 구조

### 4-1. 토큰 종류

전투에서 쓰는 기본 토큰은 6종이면 충분하다.

| 토큰 | 의미 |
|------|------|
| 소리 토큰 | sound 자극 자원 |
| 온도 토큰 | temperature 자극 자원 |
| 냄새 토큰 | smell 자극 자원 |
| 행동 토큰 | behavior 자극 자원 |
| 공감 토큰 | 조커 자원, 아무 감각 1개 대체 |
| 불안 토큰 | 적이 도망치려는 압박 |

권장 원칙:

- 앞의 4개는 플레이어가 모으는 기본 자원
- `공감 토큰`은 귀한 조커
- `불안 토큰`은 적이 쌓는 패널티

### 4-2. 적 카드에 공개되는 정보

적은 전투 시작 시 아래를 공개한다.

- 선호 감각 1개
- 보조 감각 1개 또는 없음
- 거부 감각 1개 또는 없음
- 현재 교감 비용
- 불안 한계
- 현재 intent

예시:

```text
초음파 박쥐
선호: 소리
보조: 행동
거부: 온도
최종 교감 비용: 소리 4, 행동 2
불안 한계: 5
현재 intent: 경계 비행
```

이렇게 되면 플레이어는 첫 턴부터 무엇을 모아야 하는지 안다.

## 5. 전투의 승리 조건을 비용 지불형으로 바꾸기

### 5-1. 현재 구조의 문제

지금은 순화 퍼센트를 올리고, 적절한 순간 포획 확률을 굴리는 구조다.

플레이 감각은 이렇다.

- 충분히 올린 것 같은데 실패할 수 있다
- 왜 아직 부족한지 애매하다
- 스킬 카드 숫자와 최종 포획 성공이 직접 연결되지 않는다

### 5-2. 새 구조

새 모델에서는 몬스터마다 `최종 교감 비용`이 있다.

예시:

```text
교감 비용
소리 4
행동 2
```

플레이어는 전투 중 토큰을 모으고, 공감 할인과 조커를 쌓은 뒤,
비용을 정확히 내면 즉시 성공한다.

즉 승리 조건은 아래처럼 바뀐다.

- 기존: 순화가 충분할 때 포획 확률 시도
- 신규: 토큰과 할인으로 최종 비용 충족

이건 훨씬 보드게임답다.

### 5-3. 왜 이게 더 직관적인가

플레이어는 항상 아래만 보면 된다.

- 남은 필요 토큰이 무엇인가
- 지금 가진 토큰이 무엇인가
- 할인 아이콘이 무엇인가
- 불안이 한계까지 얼마나 남았는가

즉 `%`가 아니라 `남은 비용`만 보면 된다.

---

## 6. 감각 토큰 획득 규칙

### 6-1. 기본 규칙

아군 행동은 복잡한 배율 대신 아래 네 가지 문법으로 정리한다.

1. 토큰 획득
2. 토큰 변환
3. 할인 획득
4. 불안 제거

예시:

```text
부드러운 반향
소리 토큰 2 획득
선호 감각이면 공감 토큰 1 획득
```

```text
낯선 몸짓
행동 토큰 1, 냄새 토큰 1 획득
거부 감각이 있으면 불안 1 증가
```

```text
안정 호흡
불안 토큰 2 제거
공감 토큰은 얻지 않음
```

### 6-2. 스플랜더식 액션 압축

플레이어가 한 눈에 이해하려면, 토큰 획득 규칙도 스플랜더처럼 짧아야 한다.

권장 액션 타입:

- `다른 색 2개 획득`
- `같은 색 2개 획득`
- `1개 버리고 2개 얻기`
- `조커 1개 얻기`
- `불안 1~2 제거`
- `할인 1개 배치`

즉 카드 텍스트가 길어지는 것을 막고, 전투 내 산수를 줄인다.

---

## 7. 영구 엔진: 공감 할인 아이콘

### 7-1. 스플랜더에서 가장 중요한 부분

스플랜더는 토큰을 잠깐 모으는 게임이 아니라,
카드를 사서 **영구 할인 엔진**을 만드는 게임이다.

전투도 이 구조를 가져와야 "그냥 토큰만 모으는 반복"이 되지 않는다.

### 7-2. 전투에서의 할인 의미

전투에서는 특정 행동이 성공하면 `공감 할인 아이콘`을 얻는다.

예시:

- 소리 관찰 성공 → `소리 할인 +1`
- 안정 행동 성공 → `불안 한계 임시 +1`
- 깊은 교감 행동 성공 → `공감 조커 할인 +1`

이 할인은 그 전투 동안 계속 유지된다.

즉 초반에는 토큰을 모으고,
중반에는 할인 엔진을 깔고,
후반에는 적은 토큰으로 최종 교감 비용을 내게 된다.

### 7-3. 왜 이 구조가 현재 게임과 잘 맞는가

현재 게임에는 이미 3인 파티, 역할군, 스킬 장착이 있다.
이 구조는 할인 엔진과 잘 맞는다.

- 서포터: 안정 할인, 조커 생성
- 스피드형: 토큰 가속
- 탱커: 불안 억제 할인
- 자극형: 선호 감각 할인

즉 몬스터 역할이 `배율`이 아니라 `엔진 파트`로 보이게 된다.

---

## 8. 공감 토큰은 골드 조커처럼 써야 한다

`공감 토큰`은 스플랜더의 골드처럼 귀하고 강해야 한다.

권장 규칙:

- 아무 감각 토큰 1개로 대체 가능
- 직접 모으기 어렵다
- 보통은 `예약`, `관찰 성공`, `지원형 스킬`로만 획득

이 토큰이 있으면 좋은 점:

- 잘못 모은 색을 완전히 버리지 않아도 된다
- 플레이가 막혔을 때 복구 루트가 생긴다
- 공감력 높은 아군의 존재감이 직관적으로 살아난다

즉 공감력은 성공률 숫자가 아니라, **조커 운용 능력**으로 읽히게 하는 편이 낫다.

---

## 9. 예약 시스템도 넣을 수 있다

스플랜더의 예약은 생각보다 중요하다.

전투에 맞게 바꾸면 아래처럼 쓸 수 있다.

### 9-1. 예약의 의미

특정 `교감 카드` 또는 `최종 교감 비용 슬롯`을 선점한다.

예약 효과:

- 공감 토큰 1 획득
- 해당 슬롯이 다음 턴까지 보장됨
- 일부 적 intent의 방해를 무시

예시:

```text
시선 맞추기
최종 교감 슬롯 예약
공감 토큰 1 획득
다음 턴 적의 방해로 토큰 1개를 잃지 않음
```

### 9-2. 왜 예약이 필요한가

지금 구조에서는 포획 실패가 매우 손해처럼 느껴지기 쉽다.
예약이 있으면 "이번 턴 승리하지 못해도 다음 턴을 준비했다"는 감각이 생긴다.

즉 실패 턴을 준비 턴으로 바꾸는 장치다.

---

## 10. 귀족 타일 대신 교감 마일스톤

스플랜더의 귀족은 엔진이 충분히 쌓이면 자동 방문한다.
전투에 그대로 옮기면 `교감 마일스톤`이 된다.

예시:

| 마일스톤 | 조건 | 보상 |
|------|------|------|
| 마음의 문틈 | 소리 할인 2 | 불안 1 제거 |
| 편안한 거리 | 할인 아이콘 총 3개 | 공감 토큰 1 |
| 신뢰의 순간 | 선호 감각 할인 3 | 최종 교감 비용 1 감소 |

이 구조는 좋다.

- 전투 중간 보상이 생긴다
- 엔진 빌드가 눈에 보인다
- 단순히 토큰만 주워 담는 게임이 되지 않는다

---

## 11. 불안은 퍼센트 게이지가 아니라 붉은 패널티 토큰으로

스플랜더식 구조를 써도 긴장감은 필요하다.
그 역할을 `불안 토큰`이 맡는다.

### 11-1. 기본 규칙

- 적 intent나 거부 감각 자극 시 `불안 토큰`이 쌓인다
- 불안 토큰이 한계치에 닿으면 적은 도주한다
- 일부 적은 불안이 많을수록 토큰 파괴나 비용 상승을 건다

예시:

- 불안 3 이상: 예약 불가
- 불안 4 이상: 거부 감각 비용 +1
- 불안 5: 즉시 도주

### 11-2. 왜 트랙보다 토큰이 좋은가

토큰은 실제로 "빨간 패널티가 쌓이고 있다"는 감각을 준다.
특히 보드게임풍 UI와 잘 맞는다.

또한 스킬도 훨씬 읽기 쉬워진다.

- `불안 2 제거`
- `불안 1 추가`
- `이번 턴 불안 추가 무시`

이 정도면 설명 없이도 의미가 전달된다.

---

## 12. 적 intent도 토큰 경제를 흔드는 방식으로 재설계

현재 적은 주로 공격을 한다.
스플랜더형 전투에서는 적이 플레이어의 토큰 계획을 흔드는 쪽이 더 잘 맞는다.

권장 intent:

- `경계`: 이번 턴 처음 얻는 토큰 1개 무효
- `발작`: 불안 2 추가
- `관찰`: 가장 많은 색 토큰 1개 봉인
- `교란`: 공감 토큰 사용 불가
- `도약`: 다음 턴 도주 조건 강화
- `빈틈`: 이번 턴 예약 시 추가 공감 1

이렇게 되면 적 행동도 "HP 감소"보다 **경제 방해**로 읽힌다.

이건 스플랜더식 경제 퍼즐과 잘 맞는다.

---

## 13. 3인 파티 구조와의 결합 방식

이 게임은 1인 1행동이 아니라 3마리 동시 파티다.
이건 오히려 장점이다.

각 아군이 맡는 역할을 아래처럼 분리할 수 있다.

### 13-1. 역할 분담 예시

`토큰 수집가`

- 선호 감각 토큰을 빠르게 모은다
- 같은 색 2개 액션이 강하다

`정리자`

- 불안을 제거한다
- 토큰 변환과 예약을 담당한다

`엔진 빌더`

- 할인 아이콘을 설치한다
- 교감 마일스톤을 앞당긴다

### 13-2. 턴 감각 예시

한 턴 예시:

1. 1번 아군: 소리 토큰 2 획득
2. 2번 아군: 행동 토큰 1 + 공감 토큰 1 획득
3. 3번 아군: 소리 할인 1 설치
4. 적 intent: 불안 1 추가

이건 UI만 잘 되면 플레이어가 즉시 이해한다.

## 14. 실제 예시: 초음파 박쥐를 스플랜더형으로 바꾸면

현재 데이터 기준으로 초음파 박쥐는 `sensoryType: ['sound']`, `personality: timid`다.
이를 스플랜더형 전투 카드로 바꾸면 아래처럼 설계할 수 있다.

```text
초음파 박쥐
선호: 소리
보조: 행동
거부: 온도
불안 한계: 5

최종 교감 비용
소리 4
행동 2

특수 규칙
거부 감각으로 토큰을 얻으면 불안 1 추가
불안 4 이상이면 공감 토큰 사용 불가
```

전투 흐름 예시:

1. 소리 토큰을 모은다
2. 행동 토큰을 보조로 맞춘다
3. 중간에 소리 할인 아이콘 1~2개를 깐다
4. 불안이 5 되기 전에 최종 비용을 낸다

플레이어는 "지금 순화가 62%인가 68%인가"를 고민하지 않는다.
대신 "소리 1개가 아직 부족하다"를 본다.

이게 훨씬 직관적이다.

## 15. 현재 스킬 데이터를 어떻게 옮길 것인가

### 15-1. 기존 필드의 새 의미

| 현재 필드 | 스플랜더형 역할 |
|------|------|
| `axis` | 생성 토큰 색 |
| `power` | 생성량 또는 할인 강도 |
| `escapeRisk` | 불안 추가량 |
| `effects` | 조커 생성, 예약, 봉인, 변환 효과 |
| `condition` | 특정 비용 슬롯에만 사용 가능 |
| `stateBonus` | 특정 마일스톤/intent에서 추가 보상 |

### 15-2. 권장 데이터 구조

```js
{
  id: 'sound-lullaby',
  category: 'gather',
  tokenGain: { sound: 2 },
  gainEmpathyIfPreferred: 1,
  anxietyGain: 0,
}
```

```js
{
  id: 'sound-bridge',
  category: 'convert',
  convert: { from: 'temperature', to: 'sound', amount: 1 },
  tokenGain: { behavior: 1 },
  anxietyGain: 0,
}
```

```js
{
  id: 'sound-capture',
  category: 'bond',
  payBondCost: true,
  anxietyGainOnFail: 1,
}
```

즉 스킬 스키마도 `얼마나 곱하느냐`가 아니라,
`무슨 토큰을 만들고, 바꾸고, 내느냐`로 재작성해야 한다.

## 16. 구현 우선순위

이 방식은 전투 전체를 바꾸므로, 한 번에 다 하지 않는 편이 좋다.

### 1단계

- 퍼센트 게이지를 숨긴다
- 적 카드에 최종 교감 비용을 표시한다
- 감각 토큰 4종 + 공감 토큰 + 불안 토큰을 UI에 추가한다

### 2단계

- 자극 스킬을 `토큰 획득형`으로 전환한다
- 포획 스킬을 `비용 지불형`으로 바꾼다
- 불안은 토큰 한계형으로 바꾼다

### 3단계

- 할인 아이콘 시스템 추가
- 예약 추가
- 교감 마일스톤 추가

### 4단계

- 적 intent를 경제 방해형으로 확장
- 몬스터별 교감 비용 패턴 차별화

## 17. 추천 최종안

이 프로젝트에 가장 맞는 스플랜더형 리뉴얼은 아래 조합이다.

1. 감각 4색 + 공감 조커 + 불안 패널티 토큰
2. 몬스터별 공개 교감 비용
3. 전투 중 쌓이는 영구 할인 아이콘
4. 예약을 통한 다음 턴 보장
5. 마일스톤으로 중간 보상 지급
6. 적 intent는 HP 압박보다 토큰 경제 방해 중심

이렇게 바꾸면 전투는 더 이상 "확률이 맞기를 기다리는 순화 퍼센트 게임"이 아니다.

대신 아래처럼 읽히는 게임이 된다.

- 적은 소리를 원한다
- 나는 지금 소리 3, 행동 1이 있다
- 소리 할인 1이 있으니 실제로는 소리 1만 더 모으면 된다
- 불안이 4라서 이번 턴 안에 끝내거나 안정 행동이 필요하다

이 문장은 플레이어가 바로 이해할 수 있다.
지금 시스템의 가장 큰 문제였던 "숫자가 많지만 손에 안 잡힘"을 해결하는 방향이다.

## 18. 한 줄 결론

정수 트랙 모델이 "보드게임처럼 계산하기 쉬운 전투"라면,
스플랜더형 토큰 모델은 한 단계 더 나아가 **"모으고, 깔고, 맞춰 내는 전투"**다.

현재 프로젝트의 `감각 상성`, `3인 파티`, `스킬 장착`, `교감 테마`와도 잘 맞는다.

특히 포획을 확률이 아니라 비용 지불로 바꿀 수 있다는 점에서,
플레이 체감 개선 폭이 가장 클 가능성이 높다.
