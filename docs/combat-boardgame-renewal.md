# 전투 수치 보드게임식 리뉴얼 제안

> 작성일: 2026-03-13
> 목적: 현재 전투의 퍼센트/배율 중심 계산을, 플레이어가 턴 중 바로 읽고 계산할 수 있는 보드게임식 정수 모델로 재설계한다.

---

## 1. 결론 먼저

현재 전투 시스템은 아이디어 자체는 좋지만, 실제 플레이 감각은 "직관적인 턴제"보다 "숨은 배율 엔진"에 가깝다.

- UI는 `%`로 보여주지만 내부 계산은 다층 배율 구조다.
- 같은 스킬도 턴마다 결과가 크게 달라져, 플레이어가 체감값을 외우기 어렵다.
- 포획은 가장 중요한 행동인데, 성공 시점이 숫자 직관이 아니라 확률 계산으로 읽힌다.

보드게임 관점에서 보면 지금 시스템의 핵심 문제는 난이도보다 **산수 형태가 읽히지 않는 것**이다.

따라서 리뉴얼 방향은 아래처럼 잡는 것이 좋다.

1. 퍼센트 게이지를 작은 정수 트랙으로 바꾼다.
2. 배율 계산을 없애고, `+1`, `+2`, `-1` 중심으로 만든다.
3. 포획은 확률형 마무리보다 조건형 마무리로 바꾼다.
4. 감정/어빌리티/상성은 "배수"가 아니라 "토큰 또는 한 줄 규칙"으로 바꾼다.

---

## 2. 현재 시스템 평가

### 2-1. 지금 전투가 복잡하게 느껴지는 진짜 이유

현재 자극 계산은 사실상 아래 구조다.

```text
자극 결과 =
스킬 위력
× 감각 상성
× 스탯 배율
× 감정 배율
× 어빌리티 배율
× 같은 축 반복 감쇠
× 같은 스킬 반복 감쇠
```

거기에 별도로 도주 증가량도 따로 계산된다.

```text
도주 변화 =
기본 위험도
× 상성 유불리
× 감정 보정
```

포획도 단순하지 않다.

```text
포획 성공률 =
현재 순화 비율
× 공감 스탯 배율
+ 감정 보너스
+ 상태 보너스
+ 어빌리티 보너스
```

즉 플레이어는 매 턴 아래를 동시에 머리에 올려야 한다.

- 이 스킬 기본 위력이 몇인가
- 지금 적 상성이 좋은가
- 내 친화/공감 스탯이 몇인가
- 현재 감정 상태가 무엇인가
- 같은 축을 몇 번 썼는가
- 같은 스킬을 직전에 썼는가
- 팀 어빌리티가 숨어서 얼마나 더해지는가
- 턴 종료 시 자연 도주가 얼마나 붙는가

이건 전략 깊이라기보다, **노출되지 않은 중간 계산 단계가 많은 상태**에 가깝다.

### 2-2. 현재 수치가 왜 손에 안 잡히는가

대표 예시는 기본 자극 스킬들이다.

- 기본 자극은 대체로 `power 8~10`, `escapeRisk 3~5`
- 유리 상성이면 `x1.5`, 불리 상성이면 `x0.5`
- 스탯 5는 `x1.0`, 스탯 8은 `x1.3`
- 같은 스킬 반복은 `x0.7`
- 같은 축을 여러 번 쓰면 추가 감쇠

예를 들면 `power 9` 자극도 실제 체감은 아래처럼 바뀐다.

- 잘 맞는 첫 사용: 약 `+14`
- 같은 스킬 반복: 약 `+10`
- 축 포화까지 걸리면: 약 `+8~9`
- 감정/어빌리티가 얹히면 다시 달라짐

플레이어 입장에서는 "이 카드가 대충 얼마나 좋은가"를 외우기 어렵다.

도주도 비슷하다.

- 표기 위험도 `4`가 실제로는 `+3`이 되기도 하고
- 상성/감정이 나쁘면 `+6`이 되기도 한다
- 턴 종료에 자연 도주가 따로 더해진다

즉 카드에 적힌 숫자가 결과 숫자가 아니다.

### 2-3. 보드게임 기준에서 특히 아쉬운 점

보드게임식 전투는 보통 아래 조건을 만족할수록 읽기 쉽다.

1. 공개 정보가 많다.
2. 계산이 더하기/빼기 중심이다.
3. 한 번의 행동이 바꾸는 숫자 범위가 작다.
4. 예외 규칙은 카드 한 줄로 끝난다.

현재 시스템은 반대로 가고 있다.

- 공개 정보보다 내부 배율이 많다.
- 곱셈이 많다.
- 같은 행동의 결과 범위가 넓다.
- 감정, 어빌리티, 반복 패널티가 따로따로 숨어 있다.

전투가 복잡한 것이 아니라, **계산 방식이 보드게임과 반대 방향**인 것이 문제다.

---

## 3. 리뉴얼 목표

이번 리뉴얼은 밸런스 조정보다 먼저, 플레이 중 사고 방식을 바꾸는 것이 핵심이다.

핵심 목표는 아래 4개다.

1. 스킬 카드만 보고 결과를 거의 예측할 수 있어야 한다.
2. 한 턴 계산은 곱셈 없이 끝나야 한다.
3. 포획 타이밍은 퍼센트 추정이 아니라 조건 확인으로 판단되어야 한다.
4. 몬스터 개성은 배율이 아니라 카드 규칙으로 느껴져야 한다.

---

## 4. 제안 모델: 두 개의 트랙 + 작은 정수

### 4-1. 퍼센트 대신 트랙

현재의 `순화 게이지`, `도주 게이지`를 아래처럼 바꾸는 것을 권장한다.

```text
교감 트랙: 0 ~ 8
불안 트랙: 0 ~ 6
```

승패 조건:

- `교감 트랙`이 목표치 이상이 되면 교감 시도 가능
- `불안 트랙`이 한계치에 닿으면 도주

적 카드 예시:

```text
초음파 박쥐
교감 목표 6
불안 한계 5
기본 공격 2
선호 자극: 소리
거부 자극: 행동
특수 규칙: 같은 자극을 연속으로 받으면 불안 +1
```

이렇게 되면 플레이어는 더 이상 `55 / 70` 같은 몬스터별 퍼센트 기준을 머리에 둘 필요가 없다.

### 4-2. 모든 행동은 최대 두 숫자만 바꾼다

각 행동은 아래 형식으로 통일한다.

```text
자극: 교감 +N / 불안 +M
안정: 가드 +N / 불안 -M
교감: 조건 충족 시 성공, 아니면 불안 +M
```

권장 범위:

- `+1`: 약한 효과
- `+2`: 표준 효과
- `+3`: 강한 효과
- `-1`, `-2`: 안정화 효과

중요한 점은 **예외 없이 카드에 적힌 수치가 실제 결과 수치**여야 한다는 것이다.

### 4-3. 자극 스킬 등급 예시

자극은 아래 3등급이면 충분하다.

| 등급 | 교감 | 불안 | 용도 |
|------|------|------|------|
| 약한 자극 | +1 | +0 | 안전한 누적 |
| 표준 자극 | +2 | +1 | 기본 선택 |
| 강한 자극 | +3 | +2 | 마무리 직전 압박 |

상성은 배율이 아니라 정수 보정으로 바꾼다.

- 선호 자극: 교감 `+1`
- 보통 자극: 변화 없음
- 거부 자극: 교감 `-1`, 불안 `+1`

이렇게 되면 계산은 항상 아래처럼 끝난다.

```text
표준 자극(+2/+1) + 선호 보정(+1) = 교감 +3, 불안 +1
강한 자극(+3/+2) + 거부 보정(-1/+1) = 교감 +2, 불안 +3
```

이 정도면 플레이어가 머리로 즉시 계산 가능하다.

### 4-4. 반복 패널티도 배율 대신 고정 패널티

현재의 `0.7`, `0.85` 같은 감쇠는 읽기 어렵다.

보드게임식으로는 아래처럼 단순화하는 것이 좋다.

- 직전 턴과 같은 스킬 반복: 교감 `-1`
- 같은 축을 이번 전투에 3번 이상 사용: 추가 교감 `-1`

예시:

```text
기본 값 교감 +2
같은 스킬 반복 -1
축 과다 사용 -1
= 이번엔 교감 +0
```

이 방식은 계산은 쉬우면서도, 반복 플레이 억제 목적은 그대로 달성한다.

## 5. 포획은 확률형이 아니라 조건형으로

### 5-1. 추천안: 결정형 교감

가장 추천하는 방식은 **포획 확률 계산 자체를 없애는 것**이다.

교감 행동 규칙:

- `교감 트랙 >= 목표치` 이고 `불안 트랙 <= 위험선`이면 즉시 성공
- 조건 미달에서 교감 시도 시 실패하고 `불안 +1` 또는 `+2`

권장 조건:

- 일반 몬스터: `교감 목표 달성 + 불안 3 이하`
- 겁 많은 몬스터: `교감 목표 달성 + 불안 2 이하`
- 완고한 몬스터: `교감 목표 +1` 필요

이 모델의 장점:

- 플레이어가 "지금 잡히는가"를 바로 안다
- 실패 이유가 명확하다
- 공감 스탯은 확률이 아니라 준비 속도로 설계 가능하다

### 5-2. 공감 스탯의 새 역할

공감 스탯은 성공률 배율 대신 아래 중 하나로 쓰는 것이 좋다.

1. 교감 조건에서 허용 불안치를 `+1` 늘린다.
2. 준비되지 않은 교감 실패 페널티를 `-1` 줄인다.
3. 전투 시작 시 `집중 토큰 1`을 가진다.

권장안은 `허용 불안치 +1`이다.

예시:

- 기본 규칙: 교감 목표 달성 + 불안 3 이하에서 성공
- 공감 등급이 높은 몬스터: 불안 4 이하에서도 성공

이건 확률 `58%`보다 훨씬 직관적이다.

### 5-3. 대안안: 정말 확률이 필요하다면 d6 한 번만

그래도 긴장감을 남기고 싶다면 퍼센트 대신 `1d6`만 쓰는 것이 좋다.

예시:

- 조건 미달: 시도 불가
- 목표 달성 시: `4+`면 성공
- 공감 높음: `3+`
- `신뢰` 상태: `+1`
- `불안` 상태: `-1`

즉 확률이 있어도 `60%`, `72%`, `81%`가 아니라 `4+`, `3+`로 읽히게 만든다.

## 6. 스탯은 "실수값"이 아니라 "단계값"으로 압축

### 6-1. 현재 스탯의 문제

지금 스탯은 2~11 같은 값을 가지지만, 실전에서는 다시 `1.0`, `1.1`, `1.3` 같은 배율로 변환된다.

이 구조는 두 번 복잡하다.

- 원래 숫자도 크고
- 실제 계산에서는 또 배율로 바뀐다

### 6-2. 제안: 스탯을 4단계 등급으로 변환

전투에서는 실제 값 대신 등급만 쓴다.

| 실제 수치 | 전투 등급 | 전투 효과 |
|-----------|-----------|-----------|
| 1~3 | 약함 | 보너스 없음 |
| 4~6 | 보통 | 관련 행동 `+1` |
| 7~9 | 강함 | 관련 행동 `+2` |
| 10+ | 특화 | 관련 행동 `+3` |

이 등급은 UI에 아이콘이나 별 개수로 보여줄 수 있다.

예시:

- 친화 강함: 자극 교감 `+2`
- 공감 보통: 교감 허용 불안 `+1`
- 인내 특화: 수비 시 가드 `+3`
- 민첩 강함: 행동 순서 우선

### 6-3. 6스탯을 그대로 둘지 여부

메타 성장용으로 6스탯을 유지하는 것은 가능하다. 다만 전투 계산에 직접 다 넣으면 다시 복잡해진다.

권장 방향:

- `친화`, `공감`, `인내`, `민첩`: 전투 핵심 4스탯
- `유대`, `직감`: 숫자 배율이 아니라 특수 규칙 해금용

예시:

- 유대 높음: 첫 협동 자극에 `+1`
- 직감 높음: 전투 시작 시 선호 자극 공개

즉 `유대 x1.2`, `직감 x1.15` 같은 방식은 버리고,
**한 줄 규칙으로 바꾸는 것**이 좋다.

## 7. 감정도 배율표 대신 상태 토큰으로

현재 감정은 공격/도주/순화/포획에 동시에 영향을 주는 배율표다. 이건 강력하지만 읽기 어렵다.

보드게임식으로는 감정을 아래 3종 정도로 줄이는 것이 좋다.

| 상태 | 효과 |
|------|------|
| 관심 | 다음 선호 자극 교감 `+1` |
| 안정 | 적 공격 `-1`, 턴 종료 불안 증가 없음 |
| 불안 | 턴 종료 불안 `+1`, 교감 시도 위험 증가 |

핵심은 감정 하나가 규칙 하나만 바꾸는 것이다.

현재처럼

- 공격도 달라지고
- 도주도 달라지고
- 순화도 달라지고
- 포획도 달라지는 구조

는 깊이는 있지만, 플레이 중 인지비용이 높다.

## 8. 제안 스킬 구조

### 8-1. 스킬 카드는 이렇게 읽혀야 한다

좋은 카드 텍스트 예시:

```text
부드러운 노래
자극
교감 +2 / 불안 +1
선호 자극이면 교감 +1
```

```text
날개 감싸기
안정
가드 +2 / 불안 -1
이번 턴 첫 공격만 막음
```

```text
조심스러운 손길
교감
교감 목표 달성 시 성공
실패 시 불안 +1
```

플레이어는 카드만 읽고 판단할 수 있어야 한다.

### 8-2. 카테고리별 설계 역할

`자극`

- 교감 트랙을 올린다
- 상성과 반복 패널티를 읽는 기본 행동

`안정`

- 가드 토큰 획득
- 불안 트랙 감소
- 강한 적 행동 대비

`교감`

- 준비되면 승리
- 준비되지 않으면 위험한 조기 시도

즉 3분류 체계는 유지하되, 계산 방식만 단순화한다.

## 9. UI도 같이 바뀌어야 한다

수학 모델을 바꾸면 UI도 같이 바뀌어야 효과가 난다.

권장 변경:

1. `% 바` 대신 칸형 트랙 사용
2. 스킬 카드에 실제 결과 수치만 표기
3. 선호/거부 자극은 아이콘으로 항상 노출
4. 교감 가능 상태는 `READY`처럼 명확히 표기
5. 턴 종료 자동 증가도 트랙 위에 미리 예고

예시:

```text
교감: [ ][ ][■][■][■][ ][ ][ ]  3 / 6
불안: [■][■][ ][ ][ ][ ]        2 / 5
```

이 표현은 보드게임 감각에 훨씬 가깝고, 모바일에서도 읽기 쉽다.

## 10. 현재 데이터에 맞춘 이행 가이드

### 10-1. 필드 매핑

현재 필드를 아래처럼 압축하면 된다.

| 현재 필드 | 신규 역할 |
|-----------|-----------|
| `tamingThreshold` | `bondGoal` |
| `escapeThreshold` | `panicLimit` |
| `power` | `bondGain` 또는 `guardGain` |
| `escapeRisk` | `panicGain` |
| `sensoryType` | `preferredAxis`, `rejectedAxis` |
| `emotionState` | `stateToken` |
| `attackPower` | `damage` |
| `ability` | 짧은 고정 규칙 |

### 10-2. 추천 변환 규칙

거친 1차 변환 기준:

- `tamingThreshold 45~55` → `bondGoal 5`
- `tamingThreshold 56~65` → `bondGoal 6`
- `tamingThreshold 66+` → `bondGoal 7`

- `escapeThreshold 55 이하` → `panicLimit 4`
- `escapeThreshold 56~70` → `panicLimit 5`
- `escapeThreshold 71+` → `panicLimit 6`

스킬도 아래처럼 자른다.

- 기존 자극 `power 6~8` → 교감 `+1`
- 기존 자극 `power 9~11` → 교감 `+2`
- 기존 자극 `power 12+` → 교감 `+3`

- 기존 `escapeRisk 1~2` → 불안 `+0`
- 기존 `escapeRisk 3~5` → 불안 `+1`
- 기존 `escapeRisk 6+` → 불안 `+2`

즉 복잡한 수치를 더 작은 단계로 압축한다.

### 10-3. 적 개성은 수치보다 규칙으로 준다

예시:

- 겁많은 적: 턴 종료 불안 `+1`
- 호기심 많은 적: 첫 선호 자극은 추가 교감 `+1`
- 완고한 적: 같은 자극 두 번 연속 사용 시 교감 `-1`
- 공격적인 적: 공격 피해 `+1`, 대신 불안 한계 `-1`

이런 식이면 개성이 숫자표가 아니라 경험으로 남는다.

## 11. 추천 최종안

지금 프로젝트에는 아래 조합이 가장 맞다.

1. `교감 0~8`, `불안 0~6`의 2트랙 구조
2. 모든 스킬 효과를 `+1`, `+2`, `+3`, `-1`, `-2`로 압축
3. 포획은 결정형 성공 조건으로 변경
4. 스탯은 전투 중 4단계 등급으로만 반영
5. 감정은 3종 상태 토큰으로 축소
6. 어빌리티는 배율이 아니라 "짧은 규칙 문장"으로 처리

이렇게 바꾸면 전투는 더 단순해지지만 얕아지지 않는다.

- 지금은 복잡한 계산이 깊이를 대신하고 있다.
- 리뉴얼 후에는 읽히는 정보와 턴 순서가 깊이를 만든다.

보드게임식 깊이는 "숨어 있는 배율"이 아니라, **드러난 정보 위에서의 선택**에서 나온다.

## 12. 바로 다음 작업 추천

실행 순서는 아래가 좋다.

1. 포획을 확률형에서 조건형으로 먼저 바꾼다.
2. 순화/도주 퍼센트를 칸형 트랙으로 바꾼다.
3. 스킬 데이터의 `power`, `escapeRisk`를 단계값으로 재정리한다.
4. 감정과 어빌리티의 배율식을 토큰/규칙형으로 축소한다.
5. 마지막에 UI 문구와 프리뷰를 새 모델에 맞춘다.

먼저 숫자를 작게 만들고, 그 다음 개성을 올리는 쪽이 맞다.
